Geometry::IVCoord

Converts a Coord stored in big-endian byte order to the same Coord stored in little-endian byte order, or vice versa.

    void IVCoord (
      Coord*  c
    );

Parameters

c
Pointer to the Coord being converted.

Remarks

This conversion is needed if we read a Coord from a binary file, written with the opposing byte order.
Note that Intel processors use the big-endian byte order, while PPCs use the little-endian method.

Requirements

Version: 1.0 or later
Header: Point2DData.h
Import Library: GeometryImp.lib [Win]
Module: Geometry.dll [WIN] GeometryLib [Mac]

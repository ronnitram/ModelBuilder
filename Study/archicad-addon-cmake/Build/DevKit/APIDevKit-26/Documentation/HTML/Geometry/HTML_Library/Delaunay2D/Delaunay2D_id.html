Delaunay triangulation functions

These functions make Delaunay triangulation. This triangulation originally applied on a set of points. But it is modified to be applied on polygons.
In trinagulation can be constraints. One type of them is the edges of the polygon: they must be in the triangulation and outside them there can not be triangulating sector. The other type of them - to not counting the points - is forced edges (a chain of them is called a ridge) between the points. They must be the part of the triangulation. Although in the specifying of the input polygon there are z coordinates the triangulation is wholly two dimensional, the z coords are only data in this sense.
The use of them is simple: TDInit, TDTringulate and last TDExit.

The TDGetHeightAtPoint and TDGetPointsAtLine can make queries on the trinagulated structures. Here get role the z coordinates.

Polygon2DData

A complete desctiption of a Graphisoft Geometry polygon.

    struct Polygon2DData {
      struct {
         unsigned long isBoundBoxValid                : 1;
         unsigned long isOrientationCorrect   : 1;
         unsigned long isSameDirEdgeLegal     : 1;
      } status;
      Box2DData       boundBox;
      long           nVertices;
      Coord**        vertices;
      GSHandle       vertexInfos;
      GSSize         sizeOfVertexInfo;
      GSHandle       defaultVertexInfo;
      GSHandle       inhEdgeInfos;
      GSSize         sizeOfInhEdgeInfo;
      GSHandle       defaultInhEdgeInfo;
      GSHandle       uniqEdgeInfos;
      GSSize         sizeOfUniqEdgeInfo;
      GSHandle       defaultUniqEdgeInfo;
      long           nArcs;
      PolyArcRec**   arcs;
      long           nContours;
      long**         contourEnds;
      GSHandle       contourInfos;
      GSSize         sizeOfContourInfo;
      GSHandle       defaultContourInfo;
      GSHandle       inhPolygonInfo;
      GSSize         sizeOfInhPolygonInfo;
      GSHandle       uniqPolygonInfo;
      GSSize         sizeOfUniqPolygonInfo;
      GSHandle       defaultUniqPolygonInfo;
    };

Members

status.isBoundBoxValid
Must be set to 1 if the polygon is changed without refreshing its boundBox.
 
status.isOrientationCorrect
Must be set to 1 if the polygon is changed without checking the orientation.
 
status.isSameDirEdgeLegal
Set to 1, if two suceeding edges with the same direction is legal
 
boundBox
The minimal bounding box of the polygon. It could be unfilled according to status.isBoundBoxValid.
 
nVertices
The number of vertices.
 
vertices
A handle containing exactly nVertices Coords that describes all contours of the polygon.
 
vertexInfos
A handle containing information attached to vertices. It must contain exactly nVertices items. You can use it, for example, to store the heights for a polygon-based mesh.
 
sizeOfVertexInfo
The size of one attached vertex information.
 
defaultVertexInfo
One piece of vertex information that has sizeOfVertexInfo size. This information will be attached to the new vertices created in operations performed later. With the aid of this field, you can easily trace the newly created vertices after complex operations.
 
inhEdgeInfos
A handle containing information attached to edges. It must contain exactly nVertices items. If an edge splits into more than one new edges in an operation, this information will be inherited without change to all new edges created. You can use it for example to store the side angles of a roof.
 
sizeOfInhEdgeInfo
The size of an attached inherited edge information.
 
defaultInhEdgeInfo
One piece of inherited edge information which has sizeOfInhEdgeInfo size. To the brand new edges (not created with split) in operations performed later this information will be attached. With the aid of this you can easily trace the newly created edges after complex operations.
 
uniqEdgeInfos
A handle containing information attached to edges. It must contain exactly nVertices items. If an edge splits into more pieces in an operation, this information will be inherited ONLY BY THE FIRST.
 
sizeOfUniqEdgeInfo
The size of one attached unique edge information.
 
defaultUniqEdgeInfo
One piece of unique edge information which has sizeOfUniqEdgeInfo size. To the brand new edges and edges created with split in later operations this information will be attached. With the aid of this you can easily trace the newly created edges after complex operations.
 
nArcs
The number of arced edges in the polygon.
 
arcs
The arc information for arced edges. The starting vertex, the destination vertex and the view-angle (deflection). This is a signed value, reflecting the orientation.
 
nContours
The number of contours the polyogn consists of.
 
contourEnds
An array which contains the index of the last vertex of the i-th contour. It must have exactly nContours items.
 
contourInfos
A handle containing information attached to contours. It must contain exactly nContours items. If a contour splits into more than one pieces in an operation, this information will be inherited by all parts.
 
sizeOfContourInfo
The size of one attached contour information.
 
defaultContourInfo
One piece of contour information which has sizeOfContourInfo size. To the contours created with split in later operations this information will be attached.
 
inhPolygonInfo
A handle containing information attached to the polygon. It is one piece. If the polygon breaks apart into more pieces, this information will be inherited by each piece.
 
sizeOfInhPolygonInfo
The size of one attached inherited polygon information.
 
uniqPolygonInfo
A handle containing information attached to the polygon. It is one piece. If the polygon breaks apart into more pieces, this information will be inherited ONLY BY THE FIRST.
 
sizeOfUniqPolygonInfo
The size of one attached unique polygon information.
 
defaultUniqPolygonInfo
One piece of contour information which has "sizeOfUniqPolygonInfo" size. To the polygons created with split in operations performed later this information will be attached.
 

Remarks

The Graphisoft Geometry polygon is an extension of the conventional polygon. It can have holes and curved edges though these curved edges can be only circle-arcs.
This polygon can contain or link to additional data for every vertex, edge, contour or for the entire polygon.
In this structure - in contrast to C/C++ practice - all indices are from [1..n], so the 0th elen is unused, but the n-th elem must exist.
Similarly to the rest of the Geometry module, the polygon must be always posively oriented by default.
The first vertex must be always repeated in the last for all contours. So in this representation, a triangle have four vertices, where the first and the last vertex is identical.
In a regular polygon the minimal distance between two neighboring vertices is MinPolygon2DDataVertexDistance.
The first contour is the main contour, and it must contain the others.
The first contour has a positive orientation, the others negative.

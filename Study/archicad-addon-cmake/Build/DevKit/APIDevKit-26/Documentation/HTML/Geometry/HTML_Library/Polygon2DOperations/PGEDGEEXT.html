PGEDGEEXT

PGEDGEEXT

Descripion of a Segment (vertex + following edge) in the Callback polygon operations.

    struct PGEDGEEXT {
      PGEDGESTATUSEXT  status;
      long             filler2;
      double           invlen;
      Coord            begPt;
      Coord            center;
      double           radius;
      double           fi1;
      double           fi2;
      GSSize           sizeOfVertexData;
      GSPtr            vertexData;
      GSPtr            defaultVertexData;
      GSSize           sizeOfInhEdgeData;
      GSPtr            inhEdgeData;
      GSPtr            defaultInhEdgeData;
      GSSize           sizeOfUniqEdgeData;
      GSPtr            uniqEdgeData;
      GSPtr            defaultUniqEdgeData;
      PGEDGEEXT
   };

Members

status
Status information of the edge. See detailed description at PGEDGESTATUSEXT.
 
filler2
Reserved. Must be zero.
 
invlen
The length of the edge or in case of curved edge: the distance of the starting and the ending vertices in inverted format: 1/length. Eg.: if the straight edge length is 2.0 the value of this field 0.5.
You can use two constants here too: InvLen_NotInited, which means that you leave to the operation to bother with this field. The other is InvLen_ZeroEdge, which means this edge has no length.
 
begPt
The starting vertex of the edge.
 
center
This field only valid if the edge is arced. The center of the arc.
 
radius
This field only valid if the edge is arced. The radius of the arc.
 
fi1
This field only valid if the edge is arced. The starting direction-angle of the arc.
 
fi2
This field only valid if the edge is arced. The ending direction-angle of the arc.
 
sizeOfVertexData
Size of attached data of the starting vertex.
 
vertexData
The attached data of the starting vertex.
 
defaultVertexData
It is only a reference! The data is pointed from the PGPOLYEXT. The default attached data of the starting vertex. Used for the new starting vertex if the edge splits.
 
sizeOfInhEdgeData
Size of attached inheriting data of the edge.
 
inhEdgeData
Attached inheriting data of the edge. It will inherit to all descendant piece of this edge.
 
defaultInhEdgeData
It is only a reference! The data is pointed from the PGPOLYEXT. The default attached inheriting data of the edge. Used for a brand-new edge. (NOT FOR SPLITS.)
 
sizeOfUniqEdgeData
Size of attached unique data of the edge.
 
uniqEdgeData
Attached unique data of the edge. It will inherit only to ONE descendant piece of this edge.
 
defaultUniqEdgeData
It is only a reference! The data is pointed from the PGPOLYEXT. The default attached unique data of the edge. Used for all descendant edges but one.
 

Remarks

There is two possibility to use the attached data.
The first is you give the attached data and its size, and the operations will copy it at the beginning and when the edge copies (for example splits). This method is safer as the second. You should use it if you give attached data which can change meanwhile the operation.
The second is you give the attached data, but give ZERO AS SIZE. In this case only the appropriate pointer (the data or the default) will copy. In this case its common that two or more pointers points to the same memory adress. So you can use it if the attached memory will be thrown away separatly. This method is quicker as the first.

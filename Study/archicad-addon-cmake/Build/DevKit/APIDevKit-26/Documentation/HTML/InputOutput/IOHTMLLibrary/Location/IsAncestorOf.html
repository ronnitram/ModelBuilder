Location::IsAncestorOf


Examines whether is ancestor-descendant relationship between two locations.

bool  IsAncestorOf (
    const Location& rightOp
) const;

Parameters

rightOp
Right operand of the "is ancestor of" operation.

Return value

Value Meaning
true this location is ancestor of rightOp
false this location is not ancestor of rightOp

 

Description

This method returns whether is this location ancestor of rightOp.

Informally, a location X is ancestor of a location Y if Y refers to a file system entry which is direct or indirect child of the file system entry referred to by X.
Formally, a location X is ancestor of a location Y if X is proper prefix of Y when full sequence of names from both of the locations are considered. Full sequence means that sequence of names from the local address is appended to the sequence of names from the network address. Proper prefix means that length of Y's full sequence of names must be greater then length of X's full sequence of names. However there is an exception to this rule: if network addresses are not equal and local address of X is not empty, then X can not be ancestor of Y anyway (for example if '|' is the network address / local address separator character and '/' is the name separator character, then "A\B|" is ancestor of "A\B\C|D" but "A\B|C" is not, although both are proper prefixes).

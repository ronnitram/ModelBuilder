File::Open


Opens (allows) access to the content of the file.

virtual GSErrCode  Open (
    OpenMode om
);

Parameters

om
Determines the way content of the file can be accessed. Possible modes are:
Value Meaning
ReadMode Content of the file can be accessed only by operations with read semantics.
WriteMode Content of the file can be accessed only by operations with write semantics.
WriteEmptyMode Same as WriteMode, except that content of the file is cleared at opening (length of the file content is set to 0).
ReadWriteMode ReadMode + WriteMode.

Important: buffering performed by the File class is disabled in this mode, so writing lot of simple types (Write (type) operations) is fairly ineffective. Furthermore block support on the output channel is disabled in this mode. Binary writing (in major chunks) is not affected.
AppendMode A special WriteMode where write operations always take place at the current end of the file content (appending). At opening file content position is automatically set to the end of the original content (thus existing data can not be overwritten).
Operations related to the random accessible property of the file content (seeking, file position query / set) are not allowed.

Return value

Value Meaning
NoError* OK
AccessDenied* operation is not allowed on this file
NoMoreOSFile* file can not be opened because there is no free OS handle
Error* unspecified error

* declared in File class scope
* declared in GS namespace, but accessible in the global scope

Description

Before content of a file can be accessed, it must be opened in some mode. (Other file attributes like dates can be manipulated directly without surrounding open / close operations). During opening, file content position is set to the beginning of the file content (0 position), except in appending mode, where it is set to the end of the file content.

After opening, file content can be accessed or manipulated with read, write and positioning operations according to the open mode om. An opened File instance can be used as a proper input channel (GS::IChannel), output channel (GS::OChannel) or both, depending on the open mode.

After the use, access to the file content can be explicitly closed with the Close method (preferred style). Anyway, if the file content was left opened, it will be automatically closed at the destruction time. File content can be opened and closed arbitrary times.

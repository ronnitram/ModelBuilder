Folder::CreateLink


Creates a new link in the folder.

virtual GSErrCode  CreateLink (
    const Name&     linkName,
    const Location& target
);

Parameters

linkName
Name of the new link.
target
Target location of the new link.

Return value

Value Meaning
NoError* OK
TargetExists* an entry with the same name already exists
DeviceIsFull* device is full
AccessDenied* operation is not allowed on this folder
Error* unspecified error

* declared in Folder class scope
* declared in GS namespace, but accessible in the global scope

Description

This method creates a new link with the name linkName in the folder. Link will refer to the location specified by target.

Link seems to be a native file system entry type because links are automatically resolved by InputOutput module when needed. For example if a link refers to a file, then user can not read "content" of the link, rather he or she will read content of the file link is referring to. Links can refer to file system entries of arbitrary types (files, folders or links).

CreateLink fails if the folder already contains an entry (file, folder, link) with the name linkName, and on non-writeable folders.

 Windows notes:
  • If link extensions are restricted (see the RestrictLinkExtensions function) then this method will fail to create links with extension other than ".lnk".
  • This method fails on the root folder (because it is virtual and not writeable).
  • Link referring to an another link can not be created (because new link will automatically refer to the final target).
  • Link referring to the root folder can not be created.
 Macintosh notes:
  • This method fails on the root folder (because it is virtual and not writeable).
  • Link referring to the root folder can not be created.
  • Script system code of the new link is set to smSystemScript.

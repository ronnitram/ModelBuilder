FileSystem::Move


Moves the specified entry in the file system.

GSErrCode  Move (
    const Location& from,
    const Location& to,
    DataSafety      safety = Normal
);

Parameters

from
Location of the entry to be moved.
to
New (destination or target) location of the entry.
safety
Data safety level controlling scope of the operation.

Return value

Value Meaning
NoError* OK
SourceNotFound* source entry to be moved is not found
TargetExists* an entry with the target location already exists in the file system
DeviceIsFull* device is full
AccessDenied* operation is not allowed on either of parent folders or on the entry
Error* unspecified error

* declared in FileSystem class scope
* declared in GS namespace, but accessible in the global scope

Description

This method moves (relocates) the file system entry (file, folder, link) identified with its location to a new location. Source and destination parent folders can be the same (yielding the rename operation). Move operation is applied to the whole entry (together with its internal content), meaning that move operation is inherently recursive.

Move fails if the file system already contains an entry (file, folder, link) with the target location, and if either of parent folders is not writeable. If it fails during the move operation, then it will try to restore the original file system state.

Move fails on write-protected entries (read-only entries on Windows and locked entries on Macintosh) unless IgnoreWriteProtection value is passed in the safety parameter.

 Windows notes:
  • This method fails in the root folder (because it is virtual and not writeable).
  • If the source and destination volumes are different, then moving is performed with copy and delete operations.
 Macintosh notes:
  • This method fails in the root folder (because it is virtual and not writeable) except when both parent folders are the root folder, so it is possible to rename volumes.
  • If the source and destination volumes are different or there is an entry with the source's name in the destination parent folder, then moving is performed with copy and delete operations.
  • Both resource fork and data fork are moved.

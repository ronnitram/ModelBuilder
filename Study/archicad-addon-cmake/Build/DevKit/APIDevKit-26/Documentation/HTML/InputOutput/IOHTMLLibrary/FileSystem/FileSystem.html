Description

FileSystem class gathers those operations that concern (OS) file system as a whole, and not individual files or folders.

Location is the central term of the file system. It is direct address or path of file system entries (file, folder, link). Class FileSystem can be thought of primarily as an manager object, which can locate file system entries in the file system tree directly, using their unique locations (instead of creating a root folder instance first, and then recursively creating child folders until the desired entry is reached).

Most of FileSystem methods are in some way related to locations. FileSystem can supply some special locations (e.g. location of the desktop folder on the particular OS), it can return location of mount points (starting points where physical storage devices are mapped to the logical file system tree) and it can determine target location of links (there is no separate Link class in the InputOutput module). There are also some location based "shortcut" operations, which can be otherwise carried out on Folder instances created from the particular locations.

FileSystem is a singleton class, so user can not create own instances. There is a single instance named fileSystem declared in the IO namespace to be used.

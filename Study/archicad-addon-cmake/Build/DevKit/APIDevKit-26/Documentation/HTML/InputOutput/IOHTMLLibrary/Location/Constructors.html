Description

This default constructor initializes the given instance to refer to the local root of the current machine. This constructor will always succeed.
InputOutput module already contains a public default Location instance as IO::localRoot. It is declared constant.

Instances can be created with this constructor before the module initialization!


Available from version: 1.0



explicit Location (
    const char* targetPath
);

Parameters

targetPath
Text representation (as C string) of the file system address (path) referring to the desired target.
It can be current platform's text representation or URL representation. URL representation uses the nonexisting "lan:" protocol (LAN = Local Area Network) with the following syntax:

    "lan://" networkAddress "/" localAddress

        networkAddress = (networkName ("." networkName)*)? | "localhost"

        localAddress   = (localName ("/" localName)*)?

'*' means 0 or more, '?' means 0 or 1 (optional) and '|' means or. networkName can contain any character but '.', while localName can contain any character but '/'.
Note that in URL network address (host) is represented in reverse order, that is first part from the left is the lowest order name (in contrast to the local address which is represented in "normal" order, that is first part from the left is the highest order name).
For example: "lan://Server.Subnetwork/Share/Folder/File.ext".

Description

This constructor initializes the given instance to refer to the target specified by targetPath. The target doesn't need to exist. If the targetPath is syntactically illegal (or the memory has been exhausted), then status of the Location instance becomes invalid.

Result of the construction can be examined by a subsequent invocation of the GetStatus method, where return value NoError indicates successful construction.

Instances should not be created with this constructor before the module initialization!

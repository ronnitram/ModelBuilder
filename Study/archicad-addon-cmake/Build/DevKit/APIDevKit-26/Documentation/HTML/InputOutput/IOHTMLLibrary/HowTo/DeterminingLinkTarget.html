Determining target of a link



There is no separate Link class in the InputOutput module, because of its little practical role.

New link (as a Name) with the given target (as a Location) can be created in a folder, using CreateLink method of the class Folder (it is a practical shortcut to creating an empty link in the folder, creating a Link instance and setting its target). There is also a Folder method (GetType) to determine whether is the given file system entry (as a Name) a link.

Determining target of a link is responsibility of the class FileSystem. It can be done by invoking ResolveLocation method on the single FileSystem instance named fileSystem. Input of this method is location of the link, output is target of the link as a location. If input location is not a link, then output will be set to the input (that is no effective resolvation will be performed). For example:



#include "FileSystem.hpp"    // brings everything needed to use class FileSystem (e.g. Location)

IO::Location link = GetALink ();    // obtaining a (link) location somehow
IO::Location target;                // it will hold target of the link
GSErrCode errorCode = IO::fileSystem.ResolveLocation (link, &target);    // resolves the link

 

To this sample work, InputOutput module must be initialized (=> more information).

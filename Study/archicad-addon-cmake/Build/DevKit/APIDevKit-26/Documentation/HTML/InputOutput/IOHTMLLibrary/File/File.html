Description

File is a channel (both input and output) which has file system of the particular OS behind as data source and data destination. In order to create a File instance, location (address, path) of the desired file in the file system must be supplied (see class Location). On the other hand, File is a permanent storage class which has few attributes to query (read) and set (write) including its content.

Most importantly File contains an arbitrary long raw data section referred to as content of the file. Before we can access this content, it must be opened in some mode with the Open method. After that, the particular File instance can be used as a proper input channel (GS::IChannel), output channel (GS::OChannel) or both, depending on the open mode. Finally, after the use file must be closed. This can be done explicitly by the user (preferred style) with the Close method, or can be left to the destructor (which will invoke the Close method if the file is left opened).

Other file attributes include length of the content, creation time, modification time and type information. These attributes can be queried and set directly without surrounding open / close operations.

Every opened file has a associated position referred to as "file content position". Read and write operations on the file content will be performed at the point indicated by this position. After a successful operation this position is incremented appropriately, so the next operation (if there was no seeking meanwhile) will be performed after the content segment operated on by the previous operation. File has random access property because file content position can be explicitly adjusted with the Seek method (except in appending mode).

Default input protocol is BinIProtocol, default output protocol is BinOProtocol.

 Macintosh notes:
  • File content means data fork, resource fork can not be accessed.

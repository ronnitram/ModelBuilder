How to use the CIO interface?


  • User should insert and compile the "CIOStub.cpp" stub file into the project.


  • The subset of the "traditional" IO interface can be used in the "CIO" namespace. This means that names of classes and methods (and their behaviour) remain the same, except that they are put into the "CIO" namespace instead of "IO" namespace, but only subset of classes and methods is available.


  • User should include the necessary header file for the particular class by prefixing the old header name with "CIO". For example if the Location class is needed, then user should include the "CIOLocation.hpp" header file (instead of "Location.hpp").


  • The corresponding subset of the documentation can be and should be used.


  • The InputOutput library (DLL) is of course needed, it must be accessible to the running program. Its initialization is performed automatically by the CIO interface.

Location::Set   (6)


Sets target of the location.

GSErrCode Set (
   const char* targetPath
);

Parameters

targetPath
Text representation (as C string) of the file system address (path) referring to the desired target.
It can be current platform's text representation or URL style representation. URL style representation uses the nonexisting "lan:" protocol (LAN = Local Area Network) with the following syntax:

    "lan://" networkAddress "/" localAddress

        networkAddress = (networkName ("." networkName)*)? | "localhost"

        localAddress   = (localName ("/" localName)*)?

'*' means 0 or more, '?' means 0 or 1 (optional) and '|' means or. networkName can contain any character but '.', while localName can contain any character but '/'.
Note that in URL network address (host) is represented in reverse order, that is first part from the left is the lowest order name (in contrast to the local address which is represented in "normal" order, that is first part from the left is the highest order name).
For example: "lan://Server.Subnetwork/Share/Folder/File.ext".

Return value

Value Meaning
NoError* OK
IllegalAddress* illegal address was passed
ErrMemoryFull* out of memory
Error* unspecified error

* declared in Location class scope
* declared in GS namespace, but accessible in the global scope

Description

This method sets the location to refer to the target specified by the targetPath. The target doesn't need to exist.
Set works properly even if the state of the instance is invalid (it will reset both content and internal status of the instance).

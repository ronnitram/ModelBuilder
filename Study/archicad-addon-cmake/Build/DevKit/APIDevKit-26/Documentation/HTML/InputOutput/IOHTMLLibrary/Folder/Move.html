Folder::Move


Moves the specified entry from this folder to another one.

virtual GSErrCode  Move (
    const Name& from,
    Folder&     target,
    const Name& to,
    DataSafety  safety = Normal
);

Parameters

from
Name of the entry (in this folder) to be moved.
target
Destination folder of the move operation.
to
New name of the moved entry in the destination folder.
safety
Data safety level controlling scope of the operation.

Return value

Value Meaning
NoError* OK
SourceNotFound* source entry to be moved is not found
TargetExists* an entry with the new name already exists in the destination folder
DeviceIsFull* device is full
AccessDenied* operation is not allowed on either of folders or on the entry
Error* unspecified error

* declared in Folder class scope
* declared in GS namespace, but accessible in the global scope

Description

This method moves (relocates) the file system entry (file, folder, link) with name from in this folder to the destination folder specified by target under the new name to. Names from and to can be equal. This folder and target folder can be also the same (yielding rename operation). Move operation is applied to the whole entry (together with its internal content), meaning that move operation is inherently recursive.

Move fails if the destination folder already contains an entry (file, folder, link) with the name to, and if either of folders is not writeable. If it fails during the move operation, then it will try to restore the original file system state.

Move fails on write-protected entries (read-only entries on Windows and locked entries on Macintosh) unless IgnoreWriteProtection value is passed in the safety parameter.

 Windows notes:
  • This method fails on the root folder (because it is virtual and not writeable).
  • If the source and destination volumes are different, then moving is performed with copy and delete operations.
 Macintosh notes:
  • This method fails on the root folder (because it is virtual and not writeable) except when both this folder and target folder are the root folder, so it is possible to rename volumes.
  • If the source and destination volumes are different or there is an entry with the name from in the destination folder, then moving is performed with copy and delete operations.
  • Both resource fork and data fork are moved.

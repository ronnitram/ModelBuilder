Folder::Copy   (3)


Copies the specified entry in this folder to the given destination.

virtual GSErrCode  Copy (
    const Name& from,
    const Name& to
);

Parameters

from
Name of the entry (in this folder) to be copied.
to
Name of the copied entry (in this folder).

Return value

Value Meaning
NoError* OK
SourceNotFound* source entry to be copied is not found
TargetExists* an entry with the new name already exists in the folder
DeviceIsFull* device is full
AccessDenied* operation is not allowed on this folder or on the entry
Error* unspecified error

* declared in Folder class scope
* declared in GS namespace, but accessible in the global scope

Description

This method copies (duplicates) the file system entry (file, folder, link) with name from under the new name to in the folder. Copy operation is applied to the whole entry (together with its internal content), meaning that copy operation is inherently recursive. Some of the entry's attributes (read-only flag on Windows and lock flag on Macintosh) are also copied.

Copy fails if the folder already contains an entry (file, folder, link) with the name to, and if the folder is not readable and writeable. If it fails during the copy operation, then it will try to restore the original file system state.

 Windows notes:
  • This method fails on the root folder (because it is virtual and not writeable).
 Macintosh notes:
  • This method fails on the root folder (because it is virtual and not writeable).
  • If the volume doesn't support the file copy operation, then the file is "manually" copied (with open, read, write and close operations).
  • Both resource fork and data fork are copied.

Available from version: 1.0



virtual GSErrCode  Copy (
    const Name& from,
    Folder&     target,
    const Name& to
);

Parameters

from
Name of the entry (in this folder) to be copied.
target
Destination folder of the copy operation.
to
Name of the copied entry in the destination folder.

Return value

Value Meaning
NoError* OK
SourceNotFound* source entry to be copied is not found
TargetExists* an entry with the new name already exists in the destination folder
DeviceIsFull* device is full
AccessDenied* operation is not allowed on either of folders or on the entry
Error* unspecified error

* declared in Folder class scope
* declared in GS namespace, but accessible in the global scope

Description

This method copies (duplicates) the file system entry (file, folder, link) with name from in this folder to the destination folder specified by target under the new name to. Names from and to can be equal. This folder and target folder also can be the same (they can refer to the same location). Copy operation is applied to the whole entry (together with its internal content), meaning that copy operation is inherently recursive. Some of the entry's attributes (read-only flag on Windows and lock flag on Macintosh) are also copied.

Copy fails if the destination folder already contains an entry (file, folder, link) with the name to, and if this folder is not readable or destination folder is not writeable. If it fails during the copy operation, then it will try to restore the original file system state.

 Windows notes:
  • This method fails if the destination (target) folder is the root folder (because it is virtual and not writeable).
 Macintosh notes:
  • This method fails if the destination (target) folder is the root folder (because it is virtual and not writeable).
  • If the volumes don't support the file copy operation, then the file is "manually" copied (with open, read, write and close operations).
  • Both resource fork and data fork are copied.

Available from version: 1.0

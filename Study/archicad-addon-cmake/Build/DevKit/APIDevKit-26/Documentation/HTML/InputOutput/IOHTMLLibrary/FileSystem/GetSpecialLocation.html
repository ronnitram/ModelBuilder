FileSystem::GetSpecialLocation


Returns special locations in the file system.

GSErrCode  GetSpecialLocation (
    SpecLocId locId,
    Location* result
) const;

Parameters

locId
Special location identifier. Possible values are:
Identifier Description
ApplicationFile the running application
ApplicationFolder parent folder of the running application
CommonDocuments common documents of all users
CommonLocalSettings common local settings of all users
CurrentFolder current folder
Desktop desktop folder
SharedComponents shared components
System system folder
TemporaryFolder folder for the temporary data
UserDocuments documents of the current user
UserHome home folder of the current user
UserLocalCache local application cache of the current user
UserLocalData local application data of the current user
UserLocalSettings local application settings of the current user
UserRoamingSettings roaming settings of the current user
 
result
On return: the requested location.

Return value

Value Meaning
NoError* OK
Error* unspecified error

* declared in GS namespace, but accessible in the global scope

Description

There are some important special locations in the file system (e.g. location of the desktop folder on the particular OS). This method returns the special location identified by locId in result. It can be assumed that appropriate entry corresponding to the location returned by this method does exist in the file system. Furthermore, folder corresponding to the TemporaryFolder special location is writeable.

Note that some of these locations are platform and/or user dependent, while others can vary between executions, so it is generally bad idea to write these locations to a permanent output and use them later in subsequent executions, especially in multiplatform environments.

Note that different special locations may refer to the same place on the actual platform.

 Macintosh notes:
  • Location corresponding to the ApplicationFolder is returned when CurrentFolder is queried.
  • If entry corresponding to the particular special location doesn't exist in the file system, then it will be automatically created.

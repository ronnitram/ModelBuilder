Location::AppendToLocal   (2)


Appends new part(s) (names) to the location's local address.

GSErrCode    AppendToLocal (
    const Name& name
);

Parameters

name
New part (name) to be appended to the location's local address. C string (char*) can be also passed instead of Name.

Return value

Value Meaning
NoError* OK
ErrMemoryFull* out of memory
Error* unspecified error

* declared in GS namespace, but accessible in the global scope

Description

This method appends name to the location's local address (as last part). Length of the local address is increased by 1.


Available from version: 1.0



GSErrCode    AppendToLocal (
    const RelativeLocation& relLoc
);

Parameters

relLoc
Relative location which content (sequence of names) should be appended to the location's local address.

Return value

Value Meaning
NoError* OK
ErrMemoryFull* out of memory
Error* unspecified error

* declared in GS namespace, but accessible in the global scope

Description

This method appends content of relLoc (sequence of names) to the location's local address (as trailing parts). Length of the local address is increased by length of relLoc (if there are no leading special names in relLoc -> see below).

Two predefined names standing in the leading positions of relative locations have special meaning and special processing when appending to locations.
Special name "." means current folder (current entry) and is skipped during appending. For example "/A/B" + "././C" = "/A/B/C".
Special name ".." means parent folder (parent entry) and has effect that the corresponding parent name is deleted during appending. For example "/A/B/C" + "../../D" = "/A/D".
Special names have special meaning only in the leading positions of relative locations (e.g. "./.././../X/Y" has four special names). They are not considered and are not processed after the first normal name (e.g. "A/B/../D" has no special name).


Available from version: 1.0

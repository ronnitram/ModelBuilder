MemoryOChannel::GetDestination


Returns pointer to the data destination buffer.

char*  GetDestination (
    OwnershipType ot = OriginalOwnership
);

Parameters

ot
Determines the ownership of the returned destination buffer. Possible values are:
Value Meaning
OriginalOwnership Ownership of the buffer is not changed. The original owner (either the MemoryOChannel instance or the user) is responsible for the deleting of the buffer.
(This is the default case.)
GrantOwnership If the buffer was previously owned by the MemoryOChannel instance then ownership is passed to the user. User is responsible for the deleting of the buffer.

Buffer should be deleted with the BMpFree function!

Return value

Pointer to the data destination buffer, or NULL if state of the instance is invalid (for example due to memory exhaustion).

Description

This method returns pointer to the data destination buffer. If the buffer was previously owned by the particular instance then ownership can be granted with the GrantOwnership value of the ot parameter, so that user becomes responsible for the deleting of the buffer. In such cases user should assume that the buffer was allocated with the BMpAll function, so it should be deleted with the BMpFree function!

This method also causes internal buffers to be flushed, so user will have all the data sent to this output channel in the destination buffer. However potential errors raised during flushing are not reported. If this causes a problem, then user can invoke the Flush method manually before invoking this method. See OChannelAdapter base class about buffering details.

Size of the data sent to this output channel can be queried with the GetDataSize method.

FileSystem::Copy


Copies the specified entry in the file system.

GSErrCode  Copy (
    const Location& from,
    const Location& to
);

Parameters

from
Location of the entry to be copied.
to
Target location of the copy.

Return value

Value Meaning
NoError* OK
SourceNotFound* source entry to be copied is not found
TargetExists* an entry with the target location already exists in the file system
DeviceIsFull* device is full
AccessDenied* operation is not allowed on either of parent folders or on the entry
Error* unspecified error

* declared in FileSystem class scope
* declared in GS namespace, but accessible in the global scope

Description

This method copies (duplicates) the file system entry (file, folder, link) identified with its location to a new location. Source and destination parent folders can be the same. Copy operation is applied to the whole entry (together with its internal content), meaning that copy operation is inherently recursive. Some of the entry's attributes (read-only flag on Windows and lock flag on Macintosh) are also copied.

Copy fails if the file system already contains an entry (file, folder, link) with the target location, and if the source parent folder is not readable or destination parent folder is not writeable. If it fails during the copy operation, then it will try to restore the original file system state.

 Windows notes:
  • This method fails if the destination parent folder is the root folder (because it is virtual and not writeable).
 Macintosh notes:
  • This method fails if the destination parent folder is the root folder (because it is virtual and not writeable).
  • If the volumes don't support the file copy operation, then the file is "manually" copied (with open, read, write and close operations).
  • Both resource fork and data fork are copied.

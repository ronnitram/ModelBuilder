Initializing and uninitializing the InputOutput module



Before the first use InputOutput module must be initialized (Init), and after the last use it must be uninitialized (End). This must happen inside the main ().

Instances of classes Location and RelativeLocation created with the default constructor, instances of classes Name and protocol implementations can be used outside ("before" and "after") the main (), for example as static global variables. (Note that their constructor will be executed before the main (), that is before the module initialization! The same is true for the destructor and module uninitialization.)

The following sample code demonstrates the proper initialization-uninitialization steps.



#include "IOMain.hpp"    // contains initialization and uninitialization functions

    // initializing the InputOutput module

if (IO::Init () != 0) {
    // error, do something
}

    // using InputOutput module

.....

    // uninitializing the InputOutput module

if (IO::End () != 0) {
    // error, do something
}

RestrictLinkExtensions


Restricts extension of links to ".lnk" on Windows.

Header: IOMain.hpp

void  RestrictLinkExtensions (
    void
);

Description

Links are automatically resolved when File or Folder instances are created from a Location. However determining whether is a file system entry a link is fairly slow on Windows, because its native file system has only files and folders, and links are files (and folders) with special content which must be examined.
Process of link examination can be considerably speeded up on Windows with the common assumption that links must have extension ".lnk". This means that a file system entry with extension other than ".lnk" is not a link (it will be treated file or folder), and a file system entry with extension ".lnk" may be a link and its content will be further examined. This is a reasonable assumption which is adopted by most of applications.

By default InputOutput module doesn't expose any restriction on the extension of links. This function instructs the Windows implementation to use the former assumption (it has no effect on other platforms). It has several consequences:

  • CreateLink method of the Folder class will fail to create links with extension other than ".lnk".
  • ResolveLocation method of the FileSystem class will examine file system entries only with extension ".lnk" for potential link resolvation.
  • GetType and GetResolvedType methods of the Folder and FileSystem classes will examine file system entries only with extension ".lnk" for potential link type.
  • File and Folder constructors will try automatic resolvation only for file system entries with extension ".lnk".
  • There are special folder links (folders with special content) on Windows which typically have no ".lnk" extension. These links will not be recognized, but such links are commonly considered and resolved only in (system) file dialogs.

Please note that this function is for optimization purposes only and may change to have less or no effect in the future, so do not rely in any way on its peculiarities and consequences.

RelativeLocation::RelativeLocation   (5)


RelativeLocation constructors.

RelativeLocation (
);

Description

This default constructor initializes the given instance to an empty relative location. (An empty relative location does not change the absolute location which it is appended to.) This constructor will always succeed.

Instances can be created with this constructor before the module initialization!


Available from version: 1.0



explicit RelativeLocation (
    const Name& name
);

Parameters

name
Single name of the new RelativeLocation instance.

Description

This constructor initializes the given instance to contain the single name specified by name. If  the memory has been exhausted, then status of the RelativeLocation instance becomes invalid.

Result of the construction can be examined by a subsequent invocation of the GetStatus method, where return value NoError indicates successful construction.

Instances can be created with this constructor before the module initialization!


Available from version: 1.0



explicit RelativeLocation (
    const char* targetPath
);

Parameters

targetPath
Text representation (as C string) of the relative file system address (relative path). It can be current platform's text representation or URL representation. URL representation uses the relative URL protocol with the following syntax:

    "rel://""/" localAddress

        localAddress   = (localName ("/" localName)*)?

'*' means 0 or more, '?' means 0 or 1 (optional) and '|' means or. localName can contain any character but '/'.
For example: "rel:///Share/Folder/File.ext".

Description

This constructor initializes the given instance to refer to the relative target specified by targetPath. If the targetPath is syntactically illegal (or the memory has been exhausted), then status of the RelativeLocation instance becomes invalid.

Result of the construction can be examined by a subsequent invocation of the GetStatus method, where return value NoError indicates successful construction.

Instances should not be created with this constructor before the module initialization!

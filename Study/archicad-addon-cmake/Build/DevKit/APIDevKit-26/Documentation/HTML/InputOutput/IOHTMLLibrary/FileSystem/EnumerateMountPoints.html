FileSystem::EnumerateMountPoints


Enumerates mount points of the file system.

GSErrCode    EnumerateMountPoints (
    EnumMPCallback ec,
    void*          userData = NULL
) const;

Parameters

ec
Enumerator callback function which will be invoked for each mount point in the file system. Its declaration is: 
typedef bool   (CCALL *EnumMPCallback) (const Location& mountPoint, void* userData);
mountPoint is the location of the particular mount point, userData is an arbitrary value passed by the user through this (EnumerateMountPoints) method. If the return value from the callback function is true, then enumeration will continue, otherwise it will be stopped (in this case return value from this method is NoError).
userData
An arbitrary value which will be passed to the callback function for the user.

Return value

Value Meaning
NoError* OK
Error* unspecified error

* declared in GS namespace, but accessible in the global scope

Description

File system tree is a logical construct. File system entries are stored on different physical (or virtual) storage devices called volumes. A volume can contain several entries, and an entry can be distributed on several volumes. In practice only content of folders are distributed. Every folder is uniquely mapped to a single volume. Its child files and child links reside on this volume. Its child folders can be mapped to the same or different volumes. Term mount point is related to folders and is defined as follows. Local root (as folder) is a mount point and is mapped to some (possibly virtual) volume. Every (sub)folder that is mapped to a volume different from the volume of its parent folder is also a mount point. In other words, mount point is a folder (or more precisely its location) where underlying volume changes.

This method enumerates mount points through the callback function ec, which is invoked for each mount point in the file system. An arbitrary value can be passed to the callback function in userData. Each invocation of the callback function supplies a location of a mount point until enumeration is over or user break is encountered. Note that (local) root is also a mount point, so it will also occur during the enumeration.

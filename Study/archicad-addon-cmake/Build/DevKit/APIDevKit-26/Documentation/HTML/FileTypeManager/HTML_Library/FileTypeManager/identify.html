FileTypeManager::Identify

Searches for the best matching FileType.

    static GeneralID Identify (
      const Location&    loc,
      GroupID            rootGroupID = RootGroup,
      void*              retVal = NULL
    );

    static USize     Identify (
      const Location&    loc,
      TypeID*            retList,
      USize              listSize,
      GroupID            rootGroupID = RootGroup,
      void**             retValArray = NULL
    );

Parameters

loc
Location of the file to be identified.
retList
Array of TypeIDs. This can be NULL.
listSize
Size of retList.
rootGroupID
Limits searching to children of this group.
retVal
Callback specific return value. The callback function can optionally use this supplied pointer to a data structure to return type specific values.
retValArray
Callback specific return value array. The callback function can optionally use this supplied pointer array to data structures to return type specific values.

Return Values

The first version returns a group if more then one equally good best matches found. The returned group will contain all the best matching types and the least not matching types. This group can be the RootGroup if no other group contains all the best matching types. If only one type fits best than it will be returned. If no type matches than UnknownType will be returned.

 

The second version returns the best matching types in the supplied TypeId array. The return value is the number of equally best matching types.

Remarks

Use this function to find the best matching types based on mach level. Do not use this function to compare a TypeID or more ids to a file, because this function needs the most processing from all identifying functions. In this case use one of the much more effective matching functions (ExactMatch, ExactMatchOwn, Match, MatchOwn). See Identify vs. Match.

 

Note that if neither the extension nor the mac type matches, this function will not call all the callback functions to identify the file, but will return UnknownType.

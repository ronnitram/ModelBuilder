DGDisableNormalUpdate

Disables normal update mechanism of partially updateable user items of a dialog.

    void DGDisableNormalUpdate (
      short  dialId
    );

Parameters

dialId
ID of the dialog or tab page. DG_TOP_MODAL specifies the topmost modal dialog. DG_ALL_DIALOGS specifies all dialogs.

Remarks

User items receive DG_MSG_UPDATE messages whenever they should be updated. If the update process of a user item takes too much time, it is advised to enable partial update for that item. If partial update is enabled, in addition to normal update messages DG sends partial update messages also automatically or for the application's requirement.

If partial update is not enabled for a user item, disabling the normal update has no effect.

If the application disables the normal update with the DGDisableNormalUpdate function the dialog will receive partial update messages only until the normal update is enabled again with the DGEnableNormalUpdate function.

Disabling the normal update for a dialog is useful in the following case for example: Assume that the application has opened many modeless windows simultaneously, each one having a partial updateable user item with graphics taking long time to calculate and redraw, and these windows need to be arranged on the screen (tiled or cascaded for example). If the normal update is not disabled the windows are arranged in the following manner: the first window is moved to its new position, resized and the whole content is redrawn. Until the redraw process lasts the size and position of the next window is unchanged. Between arranging two windows long time expires. If the normal update is disabled the first window is moved, resized and receives a partial update message. Responding to this message it can repaint its background and do some basic drawing functions without time consumption. This process continues with the next window. After all windows have their new positions and sizes set, DG sends normal update messages for each window. In this case no window arrangement happens after the first normal update message is sent.

With every update message the current invalid region is sent in the message data. If the normal update is suspended for a dialog which contains a partial updateable user item, the invalid regions which are sent with partial update messages are collected by DG. The union of the collected invalid regions are sent to the dialog callback function in a normal update message when the normal update is restored by calling the DGEnableNormalUpdate function.

This function should not be called with a tab page ID.

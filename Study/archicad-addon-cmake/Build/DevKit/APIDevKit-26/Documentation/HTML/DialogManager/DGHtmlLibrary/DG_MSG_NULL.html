DG_MSG_NULL

Modal and modeless dialogs can receive this message if there is no other higher priority message for them.

Message parameters

dialId
ID of the dialog.
 
item
Not used, always zero.
 
msgData
Not used, always zero.

Return Values

In case of modeless dialogs the return value has no effect. By modal dialogs the return value should be set according to the following table:

Value Meaning
Any positive number The modal dialog will be closed and this value will be passed to the callback function with the DG_MSG_CLOSE message.
0 The modal dialog will not be closed.

Remarks

Dialogs can receive this message and only after it was enabled by DGEnableMessage.

An application can do anything appropriate when handling a DG_MSG_NULL message in the dialog callback function. For example it can call DGGetMousePosition to retrieve the position of the mouse and the state of the mouse buttons and the modifier keys. Then, according to this information, it can change the form of the cursor over the dialog.

DG_MSG_NULL is the lowest priority message in DG. A dialog receives this message only if there is no other message for it.

Requirements

Version: DG 1.0.0 or later
Header: DG.h

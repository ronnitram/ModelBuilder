DG_MSG_GROW

This message is obsolete, and kept for backward compatibility only. In new applications the DG_MSG_RESIZE message should be used which gives more information about the resize event.

The dialog has been resized.

Message parameters

dialId
ID of the dialog.
 
item
Not used, always zero.
 
msgData
Long value containing horizontal and vertical growth's.

Return Values

Not used, should be set to zero.

Remarks

The dialog callback function receives DG_MSG_GROW messages when the user resizes the dialog or the application calls the DGGrowDialog or DGSetDialogSize functions. A dialog is resizable only if one of the hGrow, vGrow or grow flags is set in its GRC specification.

When the callback function receives the DG_MSG_GROW message, the dialog already has its new size. The horizontal and vertical growth can be retrieved from the msgData message parameter by using the DGGetHGrow and DGGetVGrow macros, respectively. Positive values mean growth, negative values mean shrink.

When handling a DG_MSG_GROW message, the application typically needs to move and/or resize dialog items according to the new size of the dialog. Use DGGrowItem, DGMoveItem and DGMoveGrowItem functions to do this task. To avoid unwanted screen flickering it is advised to enclose these function calls by the DGBeginMoveGrowItems DGEndMoveGrowItems function pair.

 Windows only:

If the fulldrag system flag is on and the user begins to resize a dialog, the callback function permanently receives DG_MSG_GROW messages until the user finishes the resizing.

Requirements

Version: DG 1.0.0 or later
Header: DG.h

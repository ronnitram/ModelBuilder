Push Button

A push button is a small rectangular dialog item that is used to perform an action. There are two basic types of push buttons in DG: text- and icon-style ones and there is one special type of the push button named as split button, where a button is combined with a popup menu.

Item type

DG_ITM_BUTTON

Subtypes

Subtype Meaning
DG_BT_ICONTEXT Push button with text and/or icon.

  
DG_BT_ICON Simple icon button.

  
DG_BT_SPLIT Text and/or icon button with a popup on the side

  
Remark:
Please note that the DG_BT_TEXT subtype for push buttons is no longer supported. The DG_BT_ICONTEXT should be used instead.

GRC Specification

    Button        x  y  dx  dy  fontSpec  [frameType]    [appearanceType]  "text"
    IconButton    x  y  dx  dy  iconID    [frameType]    [appearanceType]
    SplitButton   x  y  dx  dy  fontSpec  [appearanceType]  iconID  "text"

where x and y are the pixel coordinates of the upper left corner of the push button, dx and dy are the width and height of the control in pixels. fontSpec declares the font size and style of the button text. iconID defines the icon resource ID of the icon-style push button. If the optional frameType flag is omitted, the push button is drawn with a three-dimensional raised border. If you want a frameless push button set this flag to noFrame.

Possible values of fontSpec:

Value DG constant combination used in DGSetItemFont
ExtraSmall DG_IS_EXTRASMALL
SmallPlain DG_IS_SMALL | DG_IS_PLAIN
SmallBold DG_IS_SMALL | DG_IS_BOLD
SmallItalic DG_IS_SMALL | DG_IS_ITALIC
SmallUnderline DG_IS_SMALL | DG_IS_UNDERLINE
LargePlain DG_IS_LARGE | DG_IS_PLAIN
LargeBold DG_IS_LARGE | DG_IS_BOLD
LargeItalic DG_IS_LARGE | DG_IS_ITALIC
LargeUnderline DG_IS_LARGE | DG_IS_UNDERLINE

Possible values of the appearanceType: (only affects the push button on Macintosh)

Value Meaning
Normal This is the default. The push button has normal edge (DG_BA_NORMAL).
BevelEdge The push button has bevel edge (DG_BA_BEVEL).
RoundedBevelEdge The push button has round bevel edge (DG_BA_ROUNDEDBEVEL).

Possible values of frameType:

Value Meaning
noFrame The push button will be frameless.
frame The push button will have three-dimensional raised border.

Suggested sizes

For simple text push buttons the suggested item height is 20 pixels. Notice that on Windows, DG automatically increases the height of simple text push buttons by 4 pixels in order to make enough room for the focus rectangle.

In case of icon buttons the button size should be greater than the icon size at least by 8 pixels in both direction. It is advisable to use these values in GRC or when push buttons are created from program code.

For split buttons the button size should be greater than the icon size at least by 8 pixels in height. In that case if the button does not contain an image only text, than the suggested button height is 20 pixels. It's advisable to resize the icon to the text size. The arrow size at the right side is based on the button height. (On Mac this can only be three buttons height)

Messages

Message Meaning
DG_MSG_CLICK The push button is clicked. The message data parameter is always zero.
DG_MSG_CHANGE The selection is changed in the arrow pop up. The message data parameter is always zero.

Modification Flag

The modification flag of the push button is set when the user clicks on the button and DG generates a DG_MSG_CLICK message. Use DGModified or DGResetModified to retrieve or reset (clear) the state of the modification flag.

The flag can also be set by user actions that change the selected element in the arrow pop up and generate DG_MSG_CHANGE messages.

Remarks

When a push button is clicked by the user, the dialog callback function receives a DG_MSG_CLICK message. If the application returns a non-zero value from the callback function, the dialog will be closed. The zero return value means that the application wants to stay in the dialog. It can be noted that the user can not close a dialog by clicking a push button on a tab control page.

The text of a push button with the subtype (DG_BT_ICONTEXT) can be set in GRC. It can also be set, modified and retrieved by DGSetItemText, DGCatItemText and DGGetItemText, respectively. The text can be displayed with any type of DG fonts. The font can be specified not only in GRC, but by the DGSetItemFont function as well. Use DGGetItemFont to retrieve the font currently assigned to a push button. The icon can be assigned either by the resource ID or by the handle of the icon. To set and retrieve the icon use DGSetItemImageId or DGSetItemImage and DGGetItemImageId or DGGetItemImage, respectively. Note that the icon resource ID can not be defined in GRC for this kind of button.

The icon of an icon push button (DG_BT_ICON) can be assigned either by the resource ID or by the handle of the icon. To set and retrieve the icon of a push button use DGSetItemImageId or DGSetItemImage and DGGetItemImageId or DGGetItemImage, respectively. Note that the icon resource ID can be defined in GRC, too.

The text of a split button (DG_BT_SPLIT) can be set, modified and retrieved by DGSplitButtonSetItemText and DGSplitButtonGetItemText, respectively. The text can be displayed with any type of DG fonts. The font and the size of the text can be specified not only in GRC, but by the DGSplitButtonSetItemFont function as well. Use DGSplitButtonGetItemFont to retrieve the font or the size currently assigned to a split button. The icon of the split button can be assigned either by the resource ID or by the handle of the icon. To set and retrieve the icon of a split button use DGSplitButtonSetItemIcon and DGSplitButtonGetItemIcon respectively. Notice that icon resource ID can be defined in GRC, too. It is possible to set the alignment of a split button. For setting the type of the alignment use DGSplitButtonSetAlignment function, for getting the alignment value use DGSplitButtonGetAlignment function. The icon set to the split button will be shown on the left side of the text. If the split button is RightAligned the icon will appear on the right side of the text. The pop up menu can be managed by appending an item with DGSplitButtonAppendItem or a separator with DGSplitButtonAppendSeparator, by inserting an item with DGSplitButtonInsertItem or a separator with DGSplitButtonInsertSeparator, or by erasing an item with DGSplitButtonDeleteItem. The data of a menu item can be set or retrieved by DGSplitButtonSetItemData or DGSplitButtonGetItemData. Each menu item can be enabled, disabled with DGSplitButtonEnableItem, DGSplitButtonDisableItem or DGSplitButtonSetItemStatus, you can retrieve information about the item satus with DGSplitButtonIsItemEnabled. The pop up menu can contain not only items but separators too. To decide if a menu item is a separator or not, use DGSplitButtonIsSeparatorItem function. The number of the items contained in the splitbutton can be retrieved with DGSplitButtonGetItemCount. Flashing may be avoid during refilling the popup in the split button by disabling and enabling the drawing of the button while the menu is filled up with the corresponding popupitems with the folllowing functions: DGSplitButtonEnableDraw and DGSplitButtonDisableDraw.

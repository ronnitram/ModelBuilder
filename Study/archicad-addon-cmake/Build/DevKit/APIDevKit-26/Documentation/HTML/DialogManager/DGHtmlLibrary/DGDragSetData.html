DGDragSetData

Sets the drag and drop data with the specified format and the scope.

    bool DGDragSetData (
      DGDragFormat  format,
      void*         data,
      ULong         size,
      short         scope
    );

Parameters

format
Int32 value, usually used in it's character representation ('FILE', 'TEXT'). Used to identify the format of the data being drag dropped.
 
data
Allocated data to be associated with a drag and drop operation. The drag manager will copy this data to a private memory location, so you should free your data after this function call.
 
size
Size of the data pointed by data pointer in bytes.
 
scope
The scope of the drag and drop operation. Outside of the given scope this data is explicitly rejected. The scope can take the following values: DG_DRAG_SCOPE_ITEM, DG_DRAG_SCOPE_DIALOG, DG_DRAG_SCOPE_MODULE, DG_DRAG_SCOPE_APP, DG_DRAG_SCOPE_SYSTEM.

Return Values

Value Meaning
true The format was successfully set.
false The given format cannot be set.

Remarks

There is the possibility to set up user-defined data blocks for a drag and drop operation with DGDragSetData function. A data block has a format, which is an Int32 value, and describes the data-format of this block. This format must be unique among the added data blocks' formats, so you cannot set up a data block with an already existing format. Usually this block contains the dragged items, entities (or references to them) in the specified format.

The setting of data block can be delayed, if necessary: the DGDragSetDelayedData function should be called, instead of DGDragSetData, and after DG_DRAG_SETDELAYEDDATA submessage comes, the data block must be set with DGDragSetData.

For list views, tree views, list boxes and edit controls DG sets up data block at the beginning of the drag and drop operation. These blocks' types are: 'LVEX', 'TVEX', 'LBEX', 'TEXT'. There are some special functions, which can be used with these formats: DGDragGetItemCount, DGDragGetTreeViewItem, DGDragGetListBoxItem, DGDragGetListViewItem, DGDragModifyBaseFormat.

DGDragSetData sets up a data block with the given format. During a drag and drop operation you can set drag and drop data with a certain format only once.

Use this function when DG_DRAG_START event occurs, to set a customized drag and drop data, or when DG_DRAG_SETDELAYEDDATA event occurs, to set the delayed data.

File data can be set using the 'FILE' format, the data must be a pointer to a valid IO::Location's array, allocated by placement new.

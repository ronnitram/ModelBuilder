DGEndProcessEvents

Stops event processing for a modeless dialog or a palette.

    void DGEndProcessEvents (
      short  dialId
    );

Parameters

dialId
ID of the dialog to stop event processing.

Remarks

This function should be called before the modeless dialog or palette is destroyed to stop receiving dialog messages in the dialog callback function. Once the event processing was started with DGBeginProcessEvents, it should be stopped with the DGEndProcessEvents function before destroying the dialog.

Note: The DGModelessInit function also starts event processing.

DGEndProcessEvents first hides the modeless dialog or palette. After the dialog is hided the DG_MSG_CLOSE message with DG_CLOSEBOX as the item parameter is sent to the dialog callback function, and the dialog rect will be saved by DG. At this point the dialog is ready to be destroyed.

See DGDestroyModelessDialog and DGDestroyPalette for details how to destroy modeless dialogs and palettes.

If there is no need to separate the end of event processing and the dialog destroying, use the DGModelessClose function instead of calling both DGEndProcessEvents and DGDestroyModelessDialog. DGModelessClose performs all the tasks mentioned above in one step.

Requirements

Version: DG 2.0.0 or later
Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

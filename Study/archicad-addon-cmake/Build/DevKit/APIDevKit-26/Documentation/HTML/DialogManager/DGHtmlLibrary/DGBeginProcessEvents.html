DGBeginProcessEvents

Starts event processing for a modeless dialog or a palette.

    void DGBeginProcessEvents (
      short  dialId
    );

Parameters

dialId
ID of the dialog to start event processing.

Remarks

This function should be called after the modeless dialog or the palette is created to start receiving dialog messages in the dialog callback function. Once the event processing was started, it should be stopped with the DGEndProcessEvents function before destroying the dialog.

Note: The DGModelessClose function also stops event processing.

Before the event processing begins the last used size and position of the dialog is restored by DG. The event processing begins with sending the DG_MSG_INIT message to the dialog callback function. If the restored size of the dialog does not match the new creation size, additional DG_MSG_RESIZE messages are sent. After these messages some dialog initialization are processed and the dialog is ready to use.

See DGCreateModelessDialog, DGCreatePalette, DGDestroyModelessDialog and DGDestroyPalette for details how to create and destroy modeless dialogs and palettes.

If there is no need to separate the dialog creation and the start of event processing, use the DGModelessInit function instead of calling both DGCreateModelessDialog and DGBeginProcessEvents. DGModelessInit performs all the initialization tasks mentioned above in one step.

Requirements

Version: DG 2.0.0 or later
Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

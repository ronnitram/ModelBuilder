DGListSetDialItemOnTab

This function is obsolete, use the DGListSetDialItemOnTabField function instead.

Sets the index of a dialog item that will appear on a given tabulator field of the currently selected item in a single- or multi-selection list box.

    void DGListSetDialItemOnTab (
      short  dialId,
      short  item,
      short  tabIndex,
      short  dialItem
    );

Parameters

dialId
ID of the dialog or tab page. DG_TOP_MODAL specifies the topmost modal dialog.
 
item
1-based index of the dialog item.
 
tabIndex
1-based index of tabulator field.
 
dialItem
1-based index of the dialog item that will appear on the list box.

Remarks

Dialog items that appear on a list box are called ontab items. Not all of the DG dialog items can be used as ontab items. Only items of the following types can be referred by the dialItem parameter: Check Box, Date & Time Control, Single Line Edit Control, Pop-up Control, Progress Bar, Push Button, Radio Button, Scroll Bar, Slider Bar, Spin Control, User Control. Note that Icon and Picture dialog items are not permitted here, because icons can be inserted into a list item in a more elegant manner (see DGListSetItemText). Instead of ontab user dialog items, DG gives the possibility of owner draw list tabulator fields (see DGListSetTabUpdateFlag).

Although simple (non-push-like) check boxes, can be ontab items, it is advisable to use special built-in icons instead of them to display check boxes on list items. Possible icon IDs:

Value Meaning
DG_LIST_CHECKEDICON Checked check box icon.
DG_LIST_GRAYCHECKICON Grayed check box icon.
DG_LIST_MIXEDCHECKICON Filled check box icon, used to indicate mixed or undefined selection.
DG_LIST_FRAMELESSCHECKEDICON Frameless check box icon.
DG_LIST_UNCHECKEDICON Unchecked check box icon.

The main advantage of these check box icons is that they are transparent and sizeable. If the application uses these check boxes, it should handle the DG_MSG_CLICK message and change the check box icons when the user clicks on them. To test whether the user clicks on a check box, the application should compare the horizontal position of the click with the start and end points of the tabulator fields.

From the 2.2 version of DG not only single-selection, but multi-selection list boxes also can have ontab dialog items. The list box (item) and the ontab item (dialItem) must be on the same dialog or tab page and the ontab item must precede the list box in the resource item list (dialItem < item). A tabulator field can contain only one ontab item, which is justified in the tabulator field according to the justification flag defined by DGListSetTabData. Ontab items are vertically centered within the list item rectangle.

Ontab dialog items must be hidden by DGHideItem when handling the DG_MSG_INIT message in the dialog callback function. It is advisable to determine the height of the highest ontab item belonging to a list box (use DGGetItemRect) and set the list item height to this value plus two or four by DGListSetItemHeight. This method ensures that all of the ontab items will fit into the list item rectangle and there will be a frame of selection color around the items. Use frameless edit controls as ontab items.

Ontab items can appear on the currently selected list item only. When a previously selected list item loses the selection, DG automatically removes the ontab items from it. An ontab item is applied to the following selected list item and it stays alive until that item is deselected. This means that DGListSetDialItemOnTab should be called whenever the selection in the list box is changed and ontab item(s) should be shown on the new selection. (I.e. this routine should be call when handling DG_MSG_CHANGE messages for the list box in the dialog callback function.) Ontab items do not require more handling from the application. Other tasks associated with them (for example scrolling, moving, showing, hiding, etc.) are done by DG internally.

When handling DG_MSG_CHANGE message the application should retrieve the values of the hidden ontab items and display these values on the list item that just lost the selection. Then the new values should be transferred from the currently selected list item to the ontab items.

To remove the previously set ontab dialog item from a tabulator field call DGListSetDialItemOnTab with zero dialItem parameter.

Ontab items are considered as normal dialog items. They can be handled by the usual way with their type related DG functions.

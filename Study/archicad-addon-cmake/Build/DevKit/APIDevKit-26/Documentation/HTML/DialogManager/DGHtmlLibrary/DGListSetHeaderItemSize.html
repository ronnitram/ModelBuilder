DGListSetHeaderItemSize

Sets the size of a header item of a list box item.

    short DGListSetHeaderItemSize (
      short        dialId,
      short        item,
      short        headerItem,
      short        size
    );

Parameters

dialId
ID of the dialog or tab page. DG_TOP_MODAL specifies the topmost modal dialog.
 
item
1-based index of the dialog item.
 
headerItem
1-based index of the header item.
 
size
The size of the header item.

Return Values

The new size of the header item or zero if an error occurs.

Remarks

The header can be configured so that the number and size of the header items are either synchronized with the corresponding tabulator fields by DG or independently handled by the application. The header flags can be set and retrieved by the DGListSetHeaderFlags and DGListGetHeaderFlags functions respectively.
If the isSynchron flag is set on the list box header, the DGListSetHeaderItemSize function does not affect the header item properties.

Only those header items can be resized on which the isSizeable flag is set by the DGListSetHeaderItemFlags function.

The size of the header item cannot be set smaller by mouse interaction than the minimal size of the header item. The minimal size of a header item can be set and retrieved by the DGListSetHeaderItemMinSize and the DGListGetHeaderItemMinSize functions respectively.

The first header item starts always at 0 position. The right side of the right marginal header item overlaps always with the right side of the last tabulator field. If there is space between two adjacent tabulator fields and the isSynchron flag is set on the list box header, the margins of the header items coincide with the middle of the spaces between the tabulator fields.

If the list box has a horizontal scrollbar and a header item is being resized, all the items being to the right from this header item slips to the right or to the left and the width of the header item and the width of the list box rows are changing constantly.

If the list box does not have a horizontal scrollbar, the size of the header stays constant. In this case if the user wants to widen a header item, then to the right from the chosen item another sizeable header item has to be found of which size does not gained its minimal size yet. The found header item will be narrowed. If there is no such header item, the resize will be unsuccessful. At the same time if the user wants to narrow a header item, then to the right of the chosen item the first sizeable header item will be widened.

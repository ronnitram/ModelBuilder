DG_MSG_INIT

The dialog and its items have been created and the items need to be initialized.

Message parameters

dialId
ID of the dialog.
 
item
Not used, always zero.
 
msgData
Value Meaning
DG_POS_SAVED The position and the size of the dialog has been restored from the last saved dialog rectangle.
DG_POS_DEFAULT The dialog was resized to the size given in the GRC and the dialog has been positioned at the middle of the screen.
DG_POS_ADJUSTED The size of the dialog has been restored from the last saved dialog rectangle but the position has been adjusted because the saved position was fully or partially out of the screen.

Return Values

Not used, should be set to zero.

Remarks

When handling DG_MSG_INIT, the application should initialize the dialog items. The initialization can include among others:

  • filling list boxes, list views, tree views, pop-up controls and push menu controls with items and initializing the inserted items,
  • setting the checked state of check boxes and radio buttons,
  • setting the range and value of numeric-type edit controls, date & time controls, progress bars, scroll bars, slider bars, spin controls and user controls,
  • assigning images to icon and picture controls,
  • setting the text of text-type edit controls, rich edit controls and static texts,
  • adjusting ontab dialog items on list boxes,
  • setting the keyboard focus to a specified item, etc..

If the keyboard focus is not set explicitly, DG automatically focuses the first focusable item (if any) of the dialog.

Note that only main dialogs receive DG_MSG_INIT messages. Dialog items of possible tab pages should also be initialized at the main dialog's init message.

When the dialog callback function receives the DG_MSG_INIT message, the dialog is not shown yet.

Requirements

Version: DG 1.0.0 or later

DGGetTabPageIds

This function is obsolete, use the DGTabControlGetTabPageId function instead.

Retrieves the tab page identifiers of a tab control.

    short DGGetTabPageIds (
      short  dialId,
      short  item,
      short  *tabPageIds,
      short  maxCount
    );

Parameters

dialId
ID of the dialog. DG_TOP_MODAL specifies the topmost modal dialog.
 
item
1-based index of the dialog item.
 
tabPageIds
Pointer to buffer to receive the tab page identifiers. This parameter can be nullptr.
 
maxCount
The maximal number of tab page identifiers that can be returned in the tabPageIds buffer.

Return Values

Number of tab page identifiers copied into the tabPageIds buffer.

Remarks

If tabPageIds is nullptr, DGGetTabPageIds returns the number of tab pages of the tab control. This can be also achieved by calling the DGGetItemMaxLong function. If maxCount is less than the number of tab pages, DGGetTabPageIds returns the first maxCount tab page identifiers only. If the function parameters are incorrect, zero is returned.

Tab pages are special dialogs, because they have no dialog callback function. Messages for controls on tab pages are sent to the callback function of the main dialog (the dialog that contains the tab control). For these messages, however, the ID of the tab page subdialog is passed to the callback function in the dialId parameter instead of the ID of the main dialog. The tab page IDs passed to the callback function are usually the same as the IDs defined in GRC, but in special cases they can be different. For example, if you have a modeless dialog with a multi-page tab control, DG passes the tab page IDs given in the GRC to the callback function. Suppose, however, you want an other instance of the modeless dialog. Since in DG every dialog and tab page must have a unique ID, DG assigns internally generated unique (negative) IDs to the second dialog and its tab pages, when it creates them. Later you must refer to this dialog and its tab pages by the internally generated IDs.

You can retrieve the IDs of the tab page subdialogs of a tab control by the DGGetTabPageIds function when handling the DG_MSG_INIT message in the dialog callback function. Use DGGetTabPageParentId to retrieve the ID of the main dialog of a tab page.

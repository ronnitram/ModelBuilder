DGSetAutoCloseDelay

An alert or modal dialog usually terminates by the result of a user action, i.e. click on the OK or Cancel button, or the Close Box (Windows only). However, there is a feature in DG to close these types of dialogs automatically (without user action) if a given interval expires. DGSetAutoCloseDelay sets this auto close delay.

    void DGSetAutoCloseDelay (
      ULong  delay
    );

Parameters

delay
The interval of the auto close delay in seconds. This interval can be zero as well, that means the dialog does not appear at all.

Remarks

If the given interval expires, DG closes the current alert or modal dialog. Alert functions (DGAlert, DGResAlert, DGResAlertParam) and DGModalDialog return the 1-based index of the default button. The default button is the OK button (DG_OK). If this button is disabled, the default is the Cancel button (DG_CANCEL). If both are disabled, the default is the OK button.

The given auto close delay value applies to all DG alerts and modal dialogs until a new value is set by DGSetAutoCloseDelay. The default value is zero.

To enable or disable the automatic closing of alerts and dialogs use the DGEnableAutoClose and DGDisableAutoClose functions respectively. To retrieve the current state of the automatic closing status call the DGIsAutoCloseEnabled function. By default the automatic closing is disabled.

Requirements

Version: DG 2.0.0 or later
Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

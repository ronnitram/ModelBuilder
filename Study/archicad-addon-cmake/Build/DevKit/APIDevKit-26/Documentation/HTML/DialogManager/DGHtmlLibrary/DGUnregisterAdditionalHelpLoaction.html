DGUnregisterAdditionalHelpLocation

Unregisters an AddOn's own help engine location by it's MDID and path.

    bool DGUnregisterAdditionalHelpLocation (
      Int32         developerId,
      Int32         localId
    );

Parameters

developerId
Input parameter: The DeveloperID part of the AddOn's MDID.
 
localId
Input parameter: The LocalID part of the AddOn's MDID.

Return Values

Value Meaning
true The specified helpPath was succesfully registered. There was allready a helpPath registered with this MDID.
false The specified helpPath was succesfully registered. There wasn't any helpPath registered with this MDID.

If an AddOn has a userinterface, and it want's to provide context sensitive help information, it should register it's own help engine's path during initialisation. This is mandatory, because DG pass the help anchor information to the engine as a parameter, and the engine's responsibility to process the parameter and bring up the most context sensitive information. If there's no registered engine, DG wont't know where to pass the help anchor.

As you registered the helpPath during initialisation phase (DGRegisterAdditionalHelpLocation), you should unregister during AddOn unload phase.

Requirements

Version: DG 2.1.0.66 or later
Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

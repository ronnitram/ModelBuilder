DG_MSG_TRACK

The position of the scroll box of a scroll bar, the position of the slider of a slider bar or the value of a spin control or a user control is changed during tracking.

Message parameters

dialId
ID of the dialog or tab page.
 
item
1-based index of the dialog item.
 
msgData
msgData Meaning
DG_TSM_TRACKENTER The user has just begun to track an item.
DG_TSM_TRACKING The user is tracking an item.
DG_TSM_TRACKEXIT The user has finished to track an item.
DG_TSM_LINEUP Only at scroll bar. The user moved the scroll box up with one unit.
DG_TSM_LINELEFT Only at scroll bar. The user moved the scroll box left with one unit.
DG_TSM_LINEDOWN Only at scroll bar. The user moved the scroll box down with one unit.
DG_TSM_LINERIGHT Only at scroll bar. The user moved the scroll box right with one unit.
DG_TSM_PAGEUP Only at scroll bar. The user moved the scroll box up with one page.
DG_TSM_PAGELEFT Only at scroll bar. The user moved the scroll box left with one page.
DG_TSM_PAGEDOWN Only at scroll bar. The user moved the scroll box down with one page.
DG_TSM_PAGERIGHT Only at scroll bar. The user moved the scroll box right with one page.
DG_TSM_TOP Only at scroll bar. The user moved the scroll box to the top.
DG_TSM_LEFT Only at scroll bar. The user moved the scroll box to the left side.
DG_TSM_BOTTOM Only at scroll bar. The user moved the scroll box to the bottom.
DG_TSM_RIGHT Only at scroll bar. The user moved the scroll box to the right side.
DG_TSM_THUMBTRACK Only at scroll bar. The user is dragging the scroll box.

Return Values

Not used, should be set to zero.

Remarks

The following table summarizes the user actions that result in sending DG_MSG_TRACK messages to the dialog callback function:
Dialog item type User actions
Scroll Bar Clicking and tracking the scroll box of the scroll bar.
Clicking and holding down the (left) mouse button on one of the arrows or in the gray area of the control.
Pressing and holding down one of the direction keys when the scroll bar has the keyboard focus.
Slider Bar Clicking and tracking the slider of the slider bar.
Clicking and holding down the (left) mouse button in the control's bar area.
Pressing and holding down one of the direction keys when the slider bar has the keyboard focus.
Spin Control Clicking and holding down the (left) mouse button on one of the arrows.
User Control Clicking and tracking the client area of the control.

Receiving DG_MSG_TRACK messages allows the application to modify other controls during tracking. When handling DG_MSG_TRACK messages call DGGetItemValLong to retrieve the current value of the scroll bar, slider bar, spin control or user control.

At scroll bars, when DG_SB_NOAUTOSCROLL is set, the application (dialog callback function) should position the scroll box according to the sub-message, because DG does not do this automatically.

This table tells, which submessage will be delivered for an event.
Dialog item type Beginning tracking During tracking Finishing tracking
Scroll Bar DG_TSM_TRACKENTER If DG_SB_NOAUTOSCROLL is set, event-specific sub-message (DG_TSM_LINEUP, DG_TSM_LINELEFT, etc.), otherwise DG_TSM_TRACKING. DG_TSM_TRACKEXIT
Slider Bar,
Spin Control,
User Control		 DG_TSM_TRACKENTER DG_TSM_TRACKING DG_TSM_TRACKEXIT

DG sends DG_MSG_TRACK messages to the dialog callback function until the user releases the (left) mouse button or the pressed key. As soon as tracking is finished the callback function receives a DG_MSG_CHANGE message; the message data parameter is the scroll box position, the slider position or the control value before tracking, respectively. Note that every DG_MSG_CHANGE message is preceded by at least one DG_MSG_TRACK message.

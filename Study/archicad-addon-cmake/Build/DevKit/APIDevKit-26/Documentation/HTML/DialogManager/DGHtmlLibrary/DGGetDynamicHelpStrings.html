DGGetDynamicHelpStrings

Loads the needed help related information from a dynamic help (DHLP) resource.

    bool DGGetDynamicHelpStrings (
      GSResModule       resModule,
      GSResID           resID,
      short             index,
      GS::UniString*    toolTipStr,
      GS::UniString*    anchorStr
    );

Parameters

resModule
Input parameter: Resource module of the DHLP resource.
 
resID
Input parameter: Resource ID of the DHLP resource.
 
index
Input parameter: index of the tooltip - help anchor pair you want to get.
 
toolTipStr
Output parameter: ToolTip string as a Unicode string. This parameter can be nullptr.
 
anchorStr
Output parameter: Help anchor string as a Unicode string. This parameter can be nullptr.

Return Values

Value Meaning
true The specified tooltip - help anchor pair found in the specified resModule with the specified resID. The information was succesfully loaded into the output parameters.
false The specified tooltip - help anchor pair cannot be found in the specified resModule with the specified resID. Either the resModule is missing, or there isn't any DHLP in the resModule with the specified resID, or there isn't any item in the DHLP with the specified index.

This function is can be very useful, if you want to provide dynamic help. It is recommended to store the tooltip - help anchor pairs in a DHLP resource. When either ItemContextHelpRequested or ItemToolTipRequested event occurs, you can load the required information with this function. The format of the DHLP is very similar to the DLGH format, and described in GRC Specification of the Dialog Helps.

Remarks

Any of the output string parameters can be nullptr. You can decide if you want tooltip, help anchor, or both.

Requirements

Version: DG 2.1.0.66 or later
Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

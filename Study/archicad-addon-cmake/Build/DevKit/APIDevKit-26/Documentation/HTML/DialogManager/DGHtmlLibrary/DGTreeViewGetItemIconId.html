DGTreeViewGetItemIconId

Retrieves the icon ID of the state or the normal icon of a tree view item.

    short DGTreeViewGetItemIconId (
      short  dialId,
      short  item,
      Int32  treeItem,
      short  iconType
    );

Parameters

dialId
ID of the dialog or tab page. DG_TOP_MODAL specifies the topmost modal dialog.
 
item
1-based index of the dialog item.
 
treeItem
ID of the tree view item.
 
iconType
The type of the queried icon of the tree view item:
Value Meaning
DG_TVIT_NORMAL The normal icon of the item.
DG_TVIT_STATE The state icon of the item.

Return Values

The icon ID of the specified tree view item, depending on the iconType parameter. DG_NO_ICON is returned if the specified tree view item does not contain the specified type of icon, if the icon was set by DGTreeViewSetItemIcon, or if the function parameters are incorrect.

Remarks

A tree view item can have two types of icons: the state icon and the normal icon. The icons are always on the left side of the item text. If the tree view has both types of icons, the state icon is the leftmost.

The number of different state icons in a tree view is limited to 12. There is no such limitation for the normal icons.

There are two ways to set an icon of a tree view item. One possibility is to specify the icon with its resource ID by DGTreeViewSetItemIconId or by DGTreeViewSetItemIconResource, the other is to set it with its icon handle by DGTreeViewSetItemIcon. If the icon was set by its resource ID the icon ID can be retrieved by DGTreeViewGetItemIconId, in this case DGTreeViewGetItemIcon returns nullptr.

If the icon is not set yet, an empty area is displayed instead of shifting the content of the item to the left.

Either of the icon types can be omitted by setting its size to zero either from resource or by a call to DGTreeViewSetIconSize. These settings apply to the whole tree view, i.e. to all tree items.

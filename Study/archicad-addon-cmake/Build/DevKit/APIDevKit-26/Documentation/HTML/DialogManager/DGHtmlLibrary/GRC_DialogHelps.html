GRC Specification of the Dialog Helps

Dialogs can provide context-sensitive help for their dialog items.

Help GRC Specification for any type of Dialogs

    'DLGH'  resID  DialogHelpAnchor {
      [index1]  "toolTip1"  ItemHelpAnchor1
        ...
      [indexi]  "toolTipj"  ItemHelpGroupk-ItemHelpGroupl-ItemAnchorj
        ...
      [indexk]  "toolTipn"  ItemHelpAnchorn
    }

where indexi (1 <= i <= k) is the 1-based index of a dialog item; "toolTipj" and ItemHelpAnchorj (1 <= j <= n) is the tooltip and the help anchor string of the item, respectively. Note that k can differ from n because one dialog item can have more than one tooltip and help anchor (see DGSetItemHelpIndex). The indexi parameters are optional. The GRC compiler processes the lines from the top to the bottom and if it finds a tooltip and a help string without an item index, then it uses the index in the previous line. If there is no item index in the first line then 1 is used. Note that item indices should not be regressive.

A dialog item on every type of dialog can have two types of help information: a short one and a detailed one. On Macintosh, the short displayed in one balloon help if this feature is enabled through the Macintosh system Help menu. On Windows, the usual tooltips serve to show the short item help. Tooltips are displayed only if they are enabled. When DG is initialized, tooltips are automatically enabled. Use DGDisableToolTips to disable displaying tooltips and DGEnableToolTips to enable them again.

The detailed version of the item help can be invoked by setting the keyboard focus to a focusable dialog item then pressing the [F1] key on Windows or [Help] key on Macintosh; or by clicking the right mouse button on the item; or by clicking the help [?] button in the dialog caption then clicking on the item itself (this last alternative is only avaiable on Windows platform and Modal dialogs, and if the dialog has help resource). The tooltips and the help anchor strings are defined in DLGH resources. The resource ID of the DLGH resource should be the same as the resource ID of the dialog's GDLG resource.

Help GRC Specification for dynamically generated help content purposes

    'DHLP'  resID {
      "toolTip1"  ItemHelpAnchor1
        ...
      "toolTipj"  ItemHelpGroupk-ItemHelpGroupl-ItemAnchorj
        ...
      "toolTipn"  ItemHelpAnchorn
    }

If you want to provide dynamic help information it is recommended to store the tooltip - help anchor pairs in a DHLP resource. When either ItemContextHelpRequested or ItemToolTipRequested event occurs, you can load the required information with DGGetDynamicHelpStrings function. The format of the DHLP is very similar to the DLGH format, but there is no DialogItemAnchor and there's no indexes.

DGBlankModalDialog

Displays a modal dialog. The dialog is created without resource data.

    short DGBlankModalDialog (
      short                 hSize,
      short                 vSize,
      short                 growFlag,
      short                 captionFlag,
      short                 frameFlag,
      const DGDialCallBack  dCallBack,
      DGUserData            userData
    );

Parameters

hSize
Horizontal size of the dialog.
 
vSize
Vertical size of the dialog.
 
growFlag
Flag Description
DG_DLG_NOGROW The dialog is not resizable.
DG_DLG_HGROW The dialog can be resized horizontally.
DG_DLG_VGROW The dialog can be resized vertically.
To allow both horizontal and vertical resize, use DG_DLG_HGROW | DG_DLG_VGROW as the growFlag parameter.
 
captionFlag
Flag Description
DG_DLG_NOCAPTION The dialog has no caption.
0 The dialog has a top caption.
 
frameFlag
Flag Description
DG_DLG_NOFRAME The dialog has no frame.
DG_DLG_THICKFRAME The dialog has a thick frame.
DG_DLG_NORMALFRAME The dialog has a normal frame.
 
dCallBack
Constant pointer to the dialog callback procedure.
 
userData
The user-specified data of the dialog.

Return Values

Value Meaning
0 The function parameters are incorrect or the dialog cannot be created.
A positive value A positive value was returned by the dialog callback function. This is usually the 1-based dialog item index of the clicked button.
DG_ABORT The modal dialog was aborted by DGAbortModalDialog.
 Windows only:
DG_CLOSEBOX The dialog was closed by its close box.

Remarks

DGBlankModalDialog returns only if the modal dialog is closed by the user or from program code.

The application should store the dialog ID when handling the DG_MSG_INIT message in the dialog callback function and use this value in following calls of DG functions.

DGBlankModalDialog returns only if the modal dialog is closed. The user usually closes a modal dialog by clicking the OK or the Cancel button. The dialog callback function receives a DG_MSG_CLICK message with DG_OK or DG_CANCEL as the item parameter. On Windows, if the user clicks the close box in the caption the item parameter is DG_CANCEL (or DG_OK if there is no Cancel button in the dialog). The dialog is indeed closed only if the callback function returns a non-zero value (typically DG_OK or DG_CANCEL) for the DG_MSG_CLICK message. The application can use this return value to determine whether the modal dialog completed its task successfully or was canceled by the user.

Note that a modal dialog can also be exited from program code by calling the DGAbortModalDialog function or by returning a non-zero value for a DG_MSG_NULL message in the dialog callback function. If DGAbortModalDialog is called, the callback function receives a DG_MSG_CLOSE message with DG_ABORT as the item parameter and DGBlankModalDialog returns DG_ABORT.

The dialog created with DGBlankModalDialog initially has no dialog items. Dialog items can be added with the DGAppendDialogItem function.

The user data block given by the userData parameter is passed to the dialog callback function with every DG message. The application can store dialog-specific information in this data block and cast its pointer to DGUserData.

Modal dialogs defined in GRC resource can be showed by using DGModalDialog. This function creates dialog items also.

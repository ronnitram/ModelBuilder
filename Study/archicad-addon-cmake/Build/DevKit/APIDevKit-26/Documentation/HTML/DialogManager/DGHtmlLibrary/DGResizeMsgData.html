DGResizeMsgData

If the dialog callback function receives a DG_MSG_RESIZE message, the msgData parameter is a pointer to a DGResizeMsgData structure which contains information about the resize event.

    struct DGResizeMsgData {
      short  subMessage;
      bool   userResize;
      Int8   filler;
      short  hGrow;
      short  vGrow;
    };

Members

subMessage
It can be one of the following values:
subMessage meaning
DG_RSM_RESIZEENTER Resizing of the dialog has just begun.
DG_RSM_RESIZING The dialog is being resized.
DG_RSM_RESIZEEXIT Resizing of the dialog has been finished.
DG_RSM_ABOUTTORESIZE Sent before resizing a dialog; you can constrain the dialog resize direction.
 
userResize
If the dialog or the tab page was resized by a user interface action it is true, otherwise (if the dialog was resized by the program code) it is false.
 
filler
Reserved.
 
hGrow
The horizontal grow value. Positive value means growth, negative value means shrink.
 
vGrow
The vertical grow value. Positive value means growth, negative value means shrink.

Remarks

The value of the hGrow and vGrow variables depends on the subMessage:

value meaning
DG_RSM_RESIZEENTER The hGrow and vGrow values are zero.
DG_RSM_RESIZING The hGrow and vGrow values are the current grow values.
DG_RSM_RESIZEEXIT The hGrow and vGrow values are the grow values since the DG_RSM_RESIZEENTER submessage.
DG_RSM_ABOUTTORESIZE The hGrow and vGrow values are constraints on the resize action. If you pass 0 in either of these values, the dialog won't be resized in that direction.

When handling DG_MSG_RESIZE messages, with DG_RSM_RESIZING or DG_RSM_RESIZEEXIT submessages, the application typically needs to move and/or resize dialog items according to the new size of the dialog. Use DGGrowItem, DGMoveItem and DGMoveGrowItem functions to do this task. To avoid unwanted screen flickering it is advised to enclose these function calls by the DGBeginMoveGrowItems DGEndMoveGrowItems function pair.

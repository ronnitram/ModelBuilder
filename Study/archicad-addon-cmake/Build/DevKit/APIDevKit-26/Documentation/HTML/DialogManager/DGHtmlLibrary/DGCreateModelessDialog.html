DGCreateModelessDialog

Creates a modeless dialog without showing it.

    short DGCreateModelessDialog (
      GSResModule           dialResModule,
      short                 resId,
      GSResModule           dialIconResModule,
      const DGDialCallBack  dCallBack,
      DGUserData            userData
    );

Parameters

dialResModule
The resource module the GDLG resource is in.
resId
GDLG resource identifier of the dialog.
dialIconResModule
The resource module the icons for this dialog are in.
 
dCallBack
Constant pointer to the dialog callback procedure.
 
userData
The user-specified data of the dialog.

Return Values

ID of the dialog if it was created successfully, or zero if the function parameters or the GRC description is incorrect.

Remarks

This function creates the window, the dialog items and the internal data of the modeless dialog defined in the GRC resource. It returns immediately as soon as the dialog is created. It does not show the dialog and even does not begin event processing. To receive dialog messages in the dialog callback functionthe event processing should be started with the DGBeginProcessEvents function. Use DGShowModelessDialog to show the dialog after the event processing was started.

The application should store the returned dialog ID and use this value in following calls of DG functions instead of a fixed resource ID. Basically there are two situations when this is a must:

  • You want to create more instances of a dialog (based on the same resource).
  • You want to open more dialogs of the same type that are defined in different modules with the same resource ID.

In such cases DG creates new (negative) unique dialog IDs at the second and subsequent DGCreateModelessDialog calls to solve the resource collision. It is strongly recommended to get used to this method even if the application consists of one module and every dialog is opened only once.

The user data block given by the userData parameter is passed to the dialog callback function with every DG message. The application can store dialog-specific information in this data block and cast its pointer to DGUserData.

Modeless dialogs can be created without resource by using the DGCreateBlankModelessDialog function which creates an empty dialog. To append dialog items on the created dialog use the DGAppendDialogItem function.

Requirements

Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

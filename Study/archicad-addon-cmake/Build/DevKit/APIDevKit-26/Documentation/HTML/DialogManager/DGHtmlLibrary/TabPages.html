Tab Pages

Although tab pages are not real dialogs they have similar functions as dialogs. They serve to specify dialog items on pages of normal and simple tab controls. Every tab control has a display area in which the application displays the current tab page.

A tab page can be assigned to a tab control item either in the resource script or dynamically from the program code. Use DGCreateTabPage to create a tab page which is defined in the GRC file as resource, or use DGCreateBlankTabPage to create an empty tab page on which dialog items are created with the DGAppendDialogItem function. To remove a tab item from a tab control, the tab page must be destroyed first with the DGDestroyTabPage function.

In GRC files tab pages are defined as special GDLG resources. A tab page resource contains only the dimensions of the page and the specifications of its dialog items. The application loads this resource when it creates a tab control during a DGModalDialog or DGModelessInit function call, or when the application calls the DGCreateTabPage function. Tab pages have no dialog callback function; messages for dialog items on tab pages are sent to the callback function of the main dialog (the dialog that contains the tab control). For these messages, however, the ID of the tab page subdialog is passed to the callback function in the dialId parameter instead of the ID of the main dialog.

Tab pages are placed in the display area of the tab control according to the fix (anchor) point and the grow type of the tab page. By default tab pages are centered in the display area. While resizing the tab control the fix point of the tab page keeps its place in the tab control and the size of the tab page is modified according to the grow type. The fix point and the grow type can be set and retrieved with the DGSetTabPageFixPoint, DGSetTabPageGrowType and DGGetTabPageFixPoint, DGGetTabPageGrowType functions. The size of a tab page can be modified with the DGSetTabPageSize function. Use DGGetTabPageSize to retrieve the current size. To retrieve the location and the size of a tab page in the main dialog coordinate system use the DGGetTabPageRect function. Tab pages have a minimal size which is the size given in the resource script or given in the program code in the DGCreateBlankTabPage function. This size can be modified and retrieved with the DGSetTabPageMinSize and the DGGetTabPageMinSize function.

GRC Specification

    'GDLG'  resID  TabPage  0  0  dx  dy  "" {
      dialogItem1
          ...
      dialogItemi
          ...
      dialogItemn
    }

where resID is the resource ID of the tab page. dx and dy are the width and height of the tab page in pixels. dialogItemi (1 <= i <= n) specify n pieces of dialog items. For the GRC specifications of the various dialog items see the DG dialog items page.

Help GRC Specification

Dialog items on tab pages can also have context-sensitive help information. The tab control help information can be described in a DLGH resource as in case of other dialog types. For more information about context-sensitive helps see the: GRC Specification of the Dialog Helps.

Remarks

The tab page IDs passed to the callback function of the main dialog are usually the same as the IDs defined in GRC, but in special cases they can be different. For example, if you have a modeless dialog with a multi-page tab control, DG passes the tab page IDs given in the GRC to the callback function. Suppose, however, you want an other instance of the modeless dialog. Since in DG every dialog and tab page must have a unique ID, DG assigns internally generated unique (negative) IDs to the second dialog and its tab pages, when it creates them. Later you must refer to this dialog and its tab pages by the internally generated IDs.

You can retrieve the IDs of the tab page subdialogs of a tab control by the DGTabControlGetTabPageId function when handling the DG_MSG_INIT message in the dialog callback function. Use DGGetTabPageParentId to retrieve the ID of the main dialog of a tab page.

DGContextMenuMsgData

If the dialog callback function receives a DG_MSG_CONTEXTMENU message, the msgData parameter is a pointer to a DGContextMenuMsgData structure which contains information about the context menu event.

    struct DGContextMenuMsgData {
      short  panelId;
      short  itemId;
      short  xPos;
      short  yPos;
      Int32  treeItem;
      short  listItem;
      short  listHeaderItem;
      bool   inListHeaderButton;
      bool   needHelp;
    };

Members

panelId
Id of the panel whose context help is requested.
 
itemId
1-based index of the dialog item which received the event. If the context menu event occured on the dialog, this value is zero.
 
xPos
Horizontal mouse position in screen coordinates.
 
yPos
Vertical mouse position in screen coordinates.
 
treeItem
If the dialog item is a treeview, it is the tree view item which received the event, otherwise it is zero.
 
listItem
If the dialog item is a list view or list box, it is the list item which received the event, otherwise it is zero.
 
listHeaderItem
If the dialog item is a header item of a list box item, it is the header item which received the event, otherwise it is zero.
 
inListHeaderButton
True if the dialog item is the header button of a list box item, otherwise it is false.
 
needHelp
If the user selected the "What's this"/"Help" item in the context menu, the programmer can signal the DG module to bring up the context sensitive help information for the item/dialog. So if you want DG to launch the help system with the data related to the item/dialog (this is probably because user clicked help command in the context menu) you should set this variable true. Othervise leave it untouched.

Requirements

Version: DG 2.0.0 or later

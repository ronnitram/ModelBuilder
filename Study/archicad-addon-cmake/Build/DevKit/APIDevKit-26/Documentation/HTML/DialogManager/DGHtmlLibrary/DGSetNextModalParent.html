DGSetNextModalParent

 Windows only:

Sets the parent window of the next invoked alert or modal dialog. This setting applies for the very next dialog only.

    void DGSetNextModalParent (
      const void*  parentWindow
    );

Parameters

parentWindow
Window handle (HWND) of the requested parent window cast to const void*.

Remarks

On Macintosh, this function has no effect.

On Windows, it is important to choose the right parent for a modal dialog because the user can switch to another window from an open modal dialog if that window is not an ancestor of the modal dialog.

This function must be called before a DG modal or alert dialog is opened above a system modal (Print, Open, Save, etc.) dialog to set the system modal dialog as the parent of the modal or alert dialog being opened.

The default algorithm in DG for determining the parent is the following:

  1. If there is at least one open DG modal dialog, the last one will be the parent.
  2. If there is no DG modal dialog open, the last active pop-up window is chosen. This can be a non-DG modal dialog, for example File dialogs.
  3. Otherwise the application's frame window is used.

The parent given by DGSetNextModalParent applies for the next alert function (DGAlert, DGResAlert or DGResAlertParam) or DGModalDialog call only.

To retrieve the current parent window use the DGGetNextModalParent function.

Requirements

Version: DG 2.0.0 or later
Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

DGDragModifyBaseFormat

Modifies the base drag format and the drag scope of a tree view, list view or a list box drag and drop operation.

    bool DGDragModifyBaseFormat (
      DGDragFormat  format,
      short         scope
    );

Parameters

format
Int32 value, usually used in it's character representation ('FILE', 'TEXT'). Used to identify the format of the data being drag dropped.
 
scope
The scope of the drag and drop operation. Outside of the given scope this data is explicitly rejected. The scope can take the following values: DG_DRAG_SCOPE_ITEM, DG_DRAG_SCOPE_DIALOG, DG_DRAG_SCOPE_MODULE, DG_DRAG_SCOPE_APP. Though the DG_DRAG_SCOPE_SYSTEM can be specified to, it is an invalid value for tree view, list view and list box drag and drop operations.
 

Return Values

true if the base format modification was successful, false otherwise.

Remarks

Use this function to modify the tree view 'TVEX', list view 'LVEX' or list box 'LBEX' base formats. DGDragModifyBaseFormat sets up a derived format: format, for the specified base format. These base formats are only in dialog scope. To make possible drag and drop between dialogs or even applications using the item types mentioned before, you have to modify these formats. Modifying the base format, does not effect the data itself, so all tree view, list view and list box drag and drop specific functions will operate on the modified base format data. These functions are: DGDragGetSourceDialog, DGDragGetSourceItem, DGDragGetItemCount, DGDragGetTreeViewItem, DGDragGetListBoxItem, DGDragGetListViewItem.

You should call this function when DG_DRAG_START comes, because DG sets up the drag and drop data (with the modified base format) after this for the drag source item.

GRC Specification of the Dialog Item Types

This page helps in accessing all the necessary information about the GRC specification of supported image types by the DG module.

Icon
Picture

GRC Specification of the Icon

    Icon    x  y  dx  dy  iconID  [frameType]

where x and y are the pixel coordinates of the upper left corner of the icon control, dx and dy are the width and height of the control in pixels. iconID defines the icon resource ID of the control. If the optional frameType flag is omitted, the icon is drawn without a frame.

iconID can be any positive resource ID or one of the following predefined constants:

Value Meaning Macintosh Windows
NoIcon No icon, empty space.
ErrorIcon Error icon.
InfoIcon Information icon.
WarningIcon Warning icon.

Possible values of frameType:

Value Meaning
Default The DG_FT_NONE frame type is set.
StaticEdge The DG_FT_STATIC frame type is set.
ClientEdge The DG_FT_CLIENT frame type is set.
ModalFrame The DG_FT_MODAL frame type is set.

Remarks

If an icon control is used as explanatory background for other controls, it must precede them in the dialog item list. These controls can be fully or partly within the area of the icon control.

When an icon control is created or a new icon is assigned to it from program code, the control is automatically resized to the size of the icon. During this operation the center of the control is not moved and the possible frame is taken into account.

In order to spare with resource space, always use as few colors in the color table of the icon as possible. For example if your image contains 6 different colors only, do not use an icon with 256 colors, but use one with 16 colors instead.

GRC Specification of the Picture

    Picture    x  y  dx  dy  pictID  [frameType]

where x and y are the pixel coordinates of the upper left corner of the picture control, dx and dy are the width and height of the control in pixels. On Macintosh, pictID defines the control's picture resource ID. On Windows, however, pictID refers to a bitmap resource. If the optional frameType flag is omitted, the picture is drawn without frame.

Possible values of frameType:

Value Meaning
Default The DG_FT_NONE frame type is set.
StaticEdge The DG_FT_STATIC frame type is set.
ClientEdge The DG_FT_CLIENT frame type is set.
ModalFrame The DG_FT_MODAL frame type is set.

Remarks

If a picture control is used as explanatory background for other controls, it must precede them in the dialog item list. These controls can be fully or partly within the picture control's area.

When a picture control is created or a new image is assigned to it from program code, the control is automatically resized to the size of the image. During this operation the center of the control is not moved and the possible frame is taken into account.

Although picture controls are not masked, black & white pictures seem to be transparent. The reason is that when a black & white picture is drawn, the white and black pixels are substituted with the dialog background and foreground colors, respectively.

In order to spare with resource space, always use as few colors in the image's color table as possible. For example if your image contains 6 different colors only, do not use an image with 256 colors, but use one with 16 colors instead.

Requirements

Version: DG 2.0.0 or later
Header: DG.h

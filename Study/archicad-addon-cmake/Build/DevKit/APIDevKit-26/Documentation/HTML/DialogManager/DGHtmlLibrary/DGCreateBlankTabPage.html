DGCreateBlankTabPage

Creates a tab page and assigns it to a tab control item.

    short DGCreateBlankTabPage (
      short  mainDialId,
      short  tabControlId,
      short  tabItem,
      short  hSize,
      short  vSize
    );

Parameters

mainDialId
ID of the main dialog which contains the tab control. DG_TOP_MODAL specifies the topmost modal dialog.
 
tabControlId
1-based index of the tab control dialog item.
 
tabItem
1-based index of the tab control item on which the tab page is created.
 
hSize
Horizontal size of the tab page.
 
vSize
Vertical size of the tab page.

Return Values

ID of the dialog if it was created successfully, or zero if the function parameters or the GRC description are incorrect.

Remarks

Tab pages are special dialogs. They are assigned to tab control items. When a tab control item is selected the corresponding tab page appears in the display area of the tab control.

Tab pages which are assigned to a tab control of a dialog in the GRC script are automatically created when the dialog is invoked by the application. Use this function to create and assign a tab page to an empty or a newly inserted tab control item. DGCreateBlankTabPage creates a tab page without using GRC resources. To create tab pages defined in the GRC resource use the DGCreateTabPage function. Before deleting a tab control item which has a tab page assigned, the tab page should be destroyed first with the DGDestroyTabPage function regardless of it was created from resource or from program code.

Tab pages have no dialog callback function. Messages for controls on tab pages are sent to the callback function of the main dialog. For these messages, however, the ID of the tab page subdialog is passed to the callback function in the dialId parameter instead of the ID of the main dialog.

To retrieve the ID of a tab page assigned to a tab control item, the DGTabControlGetTabPageId function can be used.

DGResAlertParam

Invokes a standard parametric DG alert given in a GALR resource.

    short DGResAlertParam (
      GSResModule  resModule,
      short                 alertId,
      const GS::UniString&  paramText1,
      const GS::UniString&  paramText2,
      const GS::UniString&  paramText3,
      const GS::UniString&  paramText4
    );

Parameters

resModule
The resource module the GALR resource is in.
alertId
GALR resource ID of the alert.
 
paramText1
First parameter text. This parameter must not be empty.
 
paramText2
Second parameter text. This parameter can be empty.
 
paramText3
Third parameter text. This parameter can be empty.
 
paramText4
Fourth parameter text. This parameter can be empty.

Return Values

Value Meaning
DG_OK The first button is clicked.
DG_CANCEL The second button is clicked.
DG_THIRD The third button is clicked.

On Windows, if the user clicks the close box in the caption, DGResAlertParam returns DG_CANCEL. If there is no second button in the alert DG_OK is returned.

If the function parameters are incorrect or the specified resource is not found, zero is returned.

Remarks

DGResAlertParam is similar to DGAlert and DGResAlert. The only difference is that its content can be specified partly in GRC, partly in run time.

The first and second alert texts are drawn using the large plain and small plain fonts, respectively. These alert texts can contain up to four "%s"format specifiers in the GALR resource. Before displaying the alert, these format specifiers are substituted with paramText1, paramText2, paramText3 and paramText4 in the order of their appearance. paramText3 can be nullptr only if both paramText1 and paramText2 are non-nullptr. paramText4 can be nullptr only if all the other text parameters are non-nullptr.

Alerts are sized automatically to the minimal size that contains all of the given elements. If the first and/or the second alert text does not fit into one line, DG automatically breaks it/them into more lines. Of course, these texts can contain explicit line breaks as well.

Requirements

Version: DG 1.0.0 or later
Header: DG.h

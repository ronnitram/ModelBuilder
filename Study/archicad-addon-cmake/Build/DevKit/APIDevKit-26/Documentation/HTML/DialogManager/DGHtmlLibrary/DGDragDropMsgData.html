DGDragDropMsgData

The DGDragDropMsgData structure contains information about a drag and drop operation.

    struct DGDragDropMsgData {
      UShort              subMessage;
      UShort              effect;
      UShort              denyFeedback;
      short               xPos;
      short               yPos;
      Int32               insertPos;
      short               relativePos;
      DGDragFormat        rqFormat;
      const void*         sourceFeedback;
      short               feedbackLeft;
      short               feedbackTop;
      short               feedbackRight;
      short               feedbackBottom;
      bool                rightDragCtxMenu;
      const GS::uchar_t*  folderPath;
    };

Members

subMessage
Reason to call.
Value Meaning
DG_DRAG_STARTING Drag source message: a drag and drop operation is about to start. If the return value on this message is 1 the drag starting is accepted. An exception is when drag is started in an edit text control, in this case 0 return value means accepted drag starting.
DG_DRAG_START Drag source message: a drag and drop operation is started, this is the moment to set your customized drag and drop data and feedback.
DG_DRAG_END Drag source message: a drag and drop operation ended.
DG_DRAG_SETDELAYEDDATA When this event rises, you can set delayed drag and drop data base on the rqFormat (requested format) value.
DG_DRAG_ENTER Drag target message: the mouse moves above an item during dragging.
DG_DRAG_LEAVE Drag target message: the mouse leaves an item during dragging.
DG_DRAG_OVER Drag target message: the mouse moves over an item during dragging.
DG_DRAG_DROP Drag target message: the item gets dropped data.
 
effect
Used as both output and input parameter. If submessage is DG_DRAG_STARTING use this field to specify the effects allowed by the source. If the current submessage is a target one, use this field to provide the necessary feedback. Can take the following values: DG_DRAG_REJECT, DG_DRAG_ACCEPT_COPY, DG_DRAG_ACCEPT_MOVE, DG_DRAG_ACCEPT_LINK, DG_DRAG_ACCEPT, DG_DRAG_DELETE. This values can be combined with bitwise OR. For example DG_DRAG_ACCEPT is the combination of DG_DRAG_ACCEPT_COPY, DG_DRAG_ACCEPT_MOVE, DG_DRAG_ACCEPT_LINK.
 
denyFeedback
Return value: Denies built-in feedback functionality, based on DG_DRAG_REJECT_SEL (target selection canceled), DG_DRAG_REJECT_INS (target insertion line canceled) values.
 
xPos
Horizontal mouse position in pixels, relative to the item client rectangle.
 
yPos
Vertical mouse position in pixels, relative to the item client rectangle.
 
insertPos
Insertion position, which can differ by item type. For example: the caret position at edit text, the item identifier at tree view, list view and list box.
 
relativePos
Relative insertion position. It is used at tree view, list view and list box controls. Relative position is based on the following values: DG_DRAG_INSERT_ON, DG_DRAG_INSERT_UNDER.
 
rqFormat
Contains the requested delayed data format when a DG_DRAG_SETDELAYEDDATA message occurs. In this case you must set the requested format with the DGDragSetData function.
 
sourceFeedback
Customized feedback data. Platform dependent native image: on Mac it must be a CGImage, on Windows HBITMAP handle is used. Used for creating user item feedback.
 
feedbackLeft
Left of the feedback image bounding rectangle.
 
feedbackTop
Top of the feedback image bounding rectangle.
 
feedbackRight
Right of the feedback image bounding rectangle.
 
feedbackBottom
Bottom of the feedback image bounding rectangle.
 
rightDragCtxMenu
Brings up the 'Copy/Move/Cancel' right drag context menu.
 
folderPath
The destination folder on a delayed 'FILE' drag and drop.

Remarks

This structure is used only as the msgData parameter of the DG_MSG_DRAGDROP message.

Requirements

Version: DG 2.0.0 or later
Header: DG.h

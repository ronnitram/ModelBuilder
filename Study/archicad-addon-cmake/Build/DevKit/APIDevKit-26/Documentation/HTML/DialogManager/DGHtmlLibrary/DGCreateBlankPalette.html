DGCreateBlankPalette

Creates a palette dialog from program code without showing it.

    short DGCreateBlankPalette (
      short                 hSize,
      short                 vSize,
      short                 growFlag,
      short                 closeFlag,
      short                 captionFlag,
      short                 frameFlag,
      const DGDialCallBack  dCallBack,
      DGUserData            userData
    );

Parameters

hSize
Horizontal size of the dialog.
 
vSize
Vertical size of the dialog.
 
growFlag
Flag Description
DG_DLG_NOGROW The dialog is not resizable.
DG_DLG_HGROW The dialog can be resized horizontally.
DG_DLG_VGROW The dialog can be resized vertically.
To allow both horizontal and vertical resize, use DG_DLG_HGROW | DG_DLG_VGROW as the growFlag parameter.
 
closeFlag
Flag Description
DG_DLG_CLOSE The dialog has a close button.
DG_DLG_NOCLOSE The dialog does not have a close button.
 
captionFlag
Flag Description
DG_DLG_NOCAPTION The dialog has no caption.
DG_DLG_TOPCAPTION The dialog has a top caption.
DG_DLG_LEFTCAPTION The dialog has a left caption.
 
frameFlag
Flag Description
DG_DLG_NOFRAME The dialog has no frame.
DG_DLG_THICKFRAME The dialog has a thick frame.
DG_DLG_NORMALFRAME The dialog has a normal frame.
 
dCallBack
Constant pointer to the dialog callback procedure.
 
userData
The user-specified data of the dialog.

Return Values

ID of the dialog if it was created successfully, or zero if the function parameters are incorrect.

Remarks

This function creates the window, the dialog items and the internal data of the palette dialog with the given parameters. It returns immediately as soon as the dialog is created. It does not show the dialog and even does not begin event processing. To receive dialog messages in the dialog callback function the event processing should be started with the DGBeginProcessEvents function. Use DGShowModelessDialog to show the dialog after the event processing was started.

The palette created with DGCreateBlankPalette or DGCreateBlankDockablePalette initially has no dialog items. Dialog items can be added with the DGAppendDialogItem function.

The application should store the returned dialog ID and use this value in following calls of DG functions.

The user data block given by the userData parameter is passed to the dialog callback function with every DG message. The application can store dialog-specific information in this data block and cast its pointer to DGUserData.

Palette dialogs defined in GRC resource can be created by using DGCreatePalette or DGCreateDockablePalette. These functions creates dialog items also.

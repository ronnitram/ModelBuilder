DG_MSG_PARTIALUPDATE

This message is obsolete, use the DG_MSG_UPDATE message instead.

A user item should be partially redrawn.

Message parameters

dialId
ID of the dialog or tab page.
 
item
1-based index of the dialog item.
 
msgData
Pointer to a DGUserMsgData structure cast to DGMessageData.

Return Values

Not used, should be set to zero.

Remarks

On Macintosh, user items always receive DG_MSG_UPDATE messages whenever it should be updated. On Windows, however, user items of DG_UT_PARTIALUPDATE subtype can receive DG_MSG_PARTIALUPDATE messages, too. This subtype can be specified in GRC. It is highly recommended to set the partial update flag of a user item if its update is a very time consuming process.

On Windows, if the full drag mode is switched on in the system, DG sends DG_MSG_PARTIALUPDATE messages while the user moves or sizes the parent dialog of the user item and sends one DG_MSG_UPDATE as soon as the user finishes moving or sizing. When the dialog callback function receives a DG_MSG_PARTIALUPDATE message the application can do a partial update, e.g. you can erase the background and draw only the frame of the item while it is resized, but when a user item receives DG_MSG_UPDATE you must redraw it entirely.

The message data parameter of the update messages is a pointer to a DGUserMsgData structure that contains all the necessary information to update the user item by the application. On Macintosh and on the emulation the drawData member of the structure should be cast to ControlHandle and the application can draw into the GrafPort pointed to by the contrlOwner member of the ControlHandle. On Windows, drawData should be cast to DRAWITEMSTRUCT* and the application can draw into the hDC member of the structure. The origin of the drawing context is set to the upper left corner of the user item.

Requirements

Version: DG 2.2.0 or later
Header: DG.h

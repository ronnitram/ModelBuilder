DGListGetHeaderFlags

Retrieves the properties of the list box header.

    void DGListGetHeaderFlags (
      short        dialId,
      short        item,
      bool*        isSynchron,
      bool*        isPushable,
      bool*        isDragable
   );

Parameters

dialId
ID of the dialog or tab page. DG_TOP_MODAL specifies the topmost modal dialog.
 
item
1-based index of the dialog item.
 
isSynchron
Output parameter: if the header items are automatically synchronized with the corresponding tabulator fields this flag is set.
 
isPushable
Output parameter: if the header items are pushable buttons this flag is set.
 
isDragable
Output parameter: if the order of the header items can be modified by mouse interaction this flag is set.

Remarks

The list box header is displayed at the top of the list box. The header can be divided into more header items. The header is used to display information about the list box columns and can be configured to enable the user to dynamically adjust the size and the order of the list box columns. The header flags can be set by the DGListSetHeaderFlags function.

If the isSynchron flag is set to true on the list box header, the properties of the header items are synchronized with the corresponding tabulator fields by DG otherwise they should be handled independently by the application. These properties are: the number and size of the header items, the font type and size, the alignment, the truncation of the text of the header items. If this flag is set, these properties are automatically updated when a property of a tabulator field is changed.
The following functions will affect the list box header items only if the isSynchron flag is set to false: DGListSetHeaderItemCount, DGListSetHeaderItemFont, DGListSetHeaderItemFlags, DGListSetHeaderItemSize.

If the isPushable flag is set to true on the list box header, DG sends a click event to the application if a header item was clicked. The DG_MSG_LISTHEADERCLICK event need to be handled by the application. Additionally DG sends the 1-based index of the clicked header item in the DGListHeaderMsgData structure.
A typical use of the pushable header items is sorting the list elements. It is possible to indicate the sorting order of a list box coloumn in a header item. Primary and secondary sorting order are shown with simple and double triangles, pointing up or downwards. To show or hide the sorting arrow the DGListSetHeaderItemArrowType function is used.

If the isDragable flag is set to true on the list box header, header items can be reordered by clicking and tracking them with the mouse. In this case DG sends a DG_MSG_LISTHEADERDRAG message to the application and additional information is sent in the DGListHeaderMsgData structure.

If any of the pointer parameters is nullptr, that value is not returned.

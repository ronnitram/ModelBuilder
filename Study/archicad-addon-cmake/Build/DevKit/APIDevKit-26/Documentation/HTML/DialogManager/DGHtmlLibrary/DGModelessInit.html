DGModelessInit

Initializes a modeless or palette dialog.

    short DGModelessInit (
      GSResModule           dialResModule,
      short                 resId,
      GSResModule           dialIconResModule,
      const DGDialCallBack  dCallBack,
      DGUserData            userData
      short                 show
    );

Parameters

dialResModule
The resource module the GDLG resource is in.
resId
GDLG resource identifier of the dialog.
dialIconResModule
The resource module the icons for this dialog are in.
 
dCallBack
Constant pointer to the dialog callback procedure.
 
userData
The user-specified data of the dialog.
 
show
Value Meaning
0 DGModelessInit does not show the dialog.
1 DGModelessInit shows the dialog.

Return Values

ID of the dialog if it was initialized successfully, or zero if the function parameters or the GRC description is incorrect.

Remarks

DGModelessInit returns immediately as soon as the dialog is created. If a modeless or palette dialog is created with zero show parameter, DGModelessInit does not show the dialog before returns. To show the dialog later use the DGShowModelessDialog function. After the dialog was created this function starts event processing also. See DGBeginProcessEvents and DGEndProcessEvents for details.

The application should store the returned dialog ID in the dialog callback function and use this value in following calls of DG functions instead of a fixed resource ID. Basically there are two situations when this is a must:

  • You want to create more instances of a dialog (based on the same resource).
  • You want to open more dialogs of the same type that are defined in different modules with the same resource ID.

In such cases DG creates new (negative) unique dialog IDs at the second and subsequent DGModelessInit calls to solve the resource collision. It is strongly recommended to get used to this method even if the application consists of one module and every dialog is opened only once.

The user data block given by the userData parameter is passed to the dialog callback function with every DG message. The application can store dialog-specific information in this data block and cast its pointer to DGUserData.

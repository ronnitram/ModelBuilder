DG_MSG_MOUSEMOVE

The mouse moves over a list box, a user item, or a static tex item.

Message parameters

dialId
ID of the dialog or tab page.
 
item
1-based index of the dialog item.
 
msgData
Item type msgData
List box Pointer to a DGListMsgData structure cast to DGMessageData.
User item Pointer to a DGUserMsgData structure cast to DGMessageData.
Static text item Pointer to a DGStaticItemMsgData structure cast to DGMessageData.

Return Values

Item type Value Meaning
List box DG_LIST_INDRAGAREA The cursor is above the drag area of a list item.
0 The cursor is not above the drag area.
User item 1 The application changes the cursor to a desired custom form.
0 The default arrow cursor is acceptable over the user item.
Static text item non-0 The application leaves the form of the cursor set by the callback.
0 The default arrow cursor is set.

Remarks

The dialog callback function can receive this message for list boxes and user items only after it was enabled by calling the DGEnableMessage function.

List items in a single-selection list box can be reordered by clicking and tracking them with the mouse. When handling the DG_MSG_MOUSEMOVE message and the cursor is above the drag area (a list tabulator field which contains an icon with the ID of DG_LIST_DRAGICON), the dialog callback function should return DG_LIST_INDRAGAREA to change the cursor to the list drag cursor (small up and down arrows). As the cursor becomes this drag cursor, the user can drag list items, i.e. change their order, by clicking and tracking the mouse. During dragging a thin dotted frame indicates the dragged item. List items can only be dragged between separator lines.

User items can receive DG_MSG_MOUSEMOVE messages when the cursor is moving over them. If you change the cursor to your custom form when handling DG_MSG_MOUSEMOVE, you must return one to prevent DG from setting back the default cursor over the user item. If you return zero DG sets the default arrow cursor.

Requirements

Version: DG 1.0.0 or later

DG_MSG_LISTUPDATE

This message is obsolete, use the DG_MSG_UPDATE message instead.

A tabulator field of a list box or the image of a list view item should be redrawn.

Message parameters

dialId
ID of the dialog or tab page.
 
item
1-based index of the dialog item.
 
msgData
Item type msgData
List box Pointer to a DGListUpdateData structure cast to DGMessageData.
List view Pointer to a DGListViewUpdateData structure cast to DGMessageData.

Return Values

Not used, should be set to zero.

Remarks

The dialog callback function receives DG_MSG_LISTUPDATE messages for a list box only if the update flag of one or more tabulator fields has been set to 1 by the DGListSetTabUpdateFlag function and these tabulator fields should be redrawn. The origin of the drawing context defined by the drawContext member of the DGListUpdateData structure is set to the upper left corner of the tabulator field. The application should just draw the desired content; graying and highlighting are handled by DG after the dialog callback function returns.

The dialog callback function receives DG_MSG_LISTUPDATE messages for a list view only if the image type of one or more list view items are DG_LVIT_OWNERDRAW and these items should be redrawn. The origin of the drawing context defined by the drawContext member of the DGListViewUpdateData structure is set to the upper left corner of the image rectangle of the list view item. Within this rectangle the application can draw the image anywhere and in arbitrary size. The application should just draw the image; graying and blending are handled by DG after the dialog callback function returns. From DG version 1.1.1, in order to blend the right area only, the application can set left and top and modify width and height members of the DGListViewUpdateData structure on return to refer the actual rectangle of the drawn image.

Requirements

Version: DG 1.0.0 (List boxes), DG 1.1.0 (List views) or later
Header: DG.h

DGImAlive

Prevents Windows to change the applications state to "Not responding".

    void DGImAlive (
      void
    );

Remarks

On Windows if an application does not send any message to the message loop for a while, Windows handles the application as it is in a 'Not responding' state. To avoid this state DGImAlive function need to be called by the application during the process lasting too long.

On Macintosh this function has no effect.

Requirements

Version: DG 2.2.0 or later
Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

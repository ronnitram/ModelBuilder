DGPostCommandToApplication

This function posts the given command into the application's message loop.

    void DGPostCommandToApplication (
      ULong        commandIdParam,
      Int32        developerIdParam,
      Int32        productIdParam,
      UIndex       multiCmdIndex
    );

Parameters

commandIdParam
The command ID to call.
 
developerIdParam
The developer ID of an add-on, if you want to call an add-on's command.
 
productIdParam
The product (local) ID of an add-on, if you want to call an add-on's command.
 
multiCmdIndex
If the command ID refers to a multi-command descriptor, then the 1-based index of the actual command to use.

Remarks

Runnable messages can be posted to the application's message loop by the DGPostCommandToApplication function. So as not to interrupt certain processes, the execution of the runnable messages can be disabled. In this case the messages are executed only after the process was finished, e.g. the modal dialog was closed. To enable or disable the execution of the runnable messages use the DGEnableRunnableMessageProcessing and DGDisableRunnableMessageProcessing functions respectively. By default the execution of the runnable messages is enabled.

DGAlert

Invokes a standard DG alert.

    short DGAlert (
      short                 alertType,
      const GS::UniString&  titleText,
      const GS::UniString&  largeText,
      const GS::UniString&  smallText,
      const GS::UniString&  button1Text,
      const GS::UniString&  button2Text,
      const GS::UniString&  button3Text
    );

Parameters

alertType
Value Meaning
DG_ERROR Stop alert on Macintosh; Error alert on Windows.
DG_WARNING Caution alert on Macintosh; Warning alert on Windows.
DG_INFORMATION Note alert on Macintosh; Information alert on Windows.
Any positive number Resource ID of a color icon to be displayed in the alert.
 
titleText
Alert title. This parameter can be empty.
 
largeText
First static text of the alert. This parameter can be empty.
 
smallText
Second static text of alert. This parameter can be empty.
 
button1Text
Text of first (right, default) button. This parameter can be empty.
 
button2Text
Text of second (middle, cancel) button. This parameter can be empty.
 
button3Text
Text of third (left) button. This parameter can be empty.

Return Values

Value Meaning
DG_OK The first button is clicked.
DG_CANCEL The second button is clicked.
DG_THIRD The third button is clicked.

If the function parameters are incorrect, zero is returned.

On Windows, if the user clicks the close box in the caption, DGAlert returns DG_CANCEL. If there is no second button in the alert DG_OK is returned.

Remarks

DGAlert has the same behavior as of DGResAlert. The only difference is in the way how the icon and the alert and button texts are specified.

If titleText is empty, there is no title on Macintosh. On Windows, in this case DG uses one of the standard alert titles given in DGInit. The first and second alert texts are drawn using the large plain and small plain fonts, respectively. If largeText or smallText is empty, there is no first or second static text. However, one of the largeText or smallText has to be non-empty. If button1Text, button2Text or button3Text is empty, there is no first, second or third button, respectively. One of the button1Text and button2Text has to be non-empty.

Alerts are sized automatically to the minimal size that contains all of the given elements. If the first and/or the second alert text does not fit into one line, DG automatically breaks it/them into more lines. Of course, these texts can contain explicit line breaks as well.

Requirements

Version: DG 1.0.0 or later
Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

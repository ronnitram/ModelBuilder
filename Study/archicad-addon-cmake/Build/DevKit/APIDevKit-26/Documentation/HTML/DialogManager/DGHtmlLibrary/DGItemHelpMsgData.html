DGItemHelpMsgData

If the dialog callback function receives a DG_MSG_ITEMHELP message for a dialog item, the msgData parameter is a pointer to a DGItemHelpMsgData structure that contains information about the help event.

    struct DGItemHelpMsgData {
      short          subMessage;
      short          itemId;
      Int32          treeItem;
      short          listItem;
      short          filler;
      Int32          tabBarItemId;
      GS::UniString  toolTipText;
      GS::UniString  contextHelpAnchor;
    };

Members

subMessage
It can be one of the following values:
subMessage meaning
DG_HSM_TOOLTIP Tooltip text is going to be displayed.
DG_HSM_CONTEXTHELP Context sensitive help is going to be displayed according to the specified help anchor.
 
itemId
1-based index of the dialog item above the message should be displayed.
 
treeItem
If the dialog item is a treeview, it is the tree view item above which the help was requested, otherwise it is zero.
 
listItem
If the help is requested above a ListView or ListBox item type, DG places 1-based index of the listItem above the help is requested.
 
filler
Reserved.
 
tabBarItemId
1-based index of the TabBar item identifier above which the help is requested.
 
toolTipText
If the subMessage is DG_HSM_TOOLTIP, the application can fill this field with a text pointer containing the message to display. This pointer must be allocated by BMAllocatePtr. It is advisable to store the corresponding tooltip and help anchor pairs in a DHLP resource. You can load the help anchor with the DGGetDynamicHelpStrings function. After the message was displayed, DG frees the allocated memory by calling the BMKillPtr function. If toolTipText is set to nullptr, DG tries to get the tooltip text from the GRC resource. On Windows platform ToolTip text can be 8192 characters long. On Windows multiple tooltips can be specified with "\n", but "\t" characters are replaced with spaces.
 
contextHelpAnchor
If the subMessage is DG_HSM_CONTEXTHELP, the application can fill this field with a text pointer containing the help anchor to be passed as parameter to the help engine registered with (DGRegisterAdditionalHelpLocation or DG::Application::RegisterHelpLocation in case of a standalone application). This pointer must be allocated by BMAllocatePtr. It is advisable to store the corresponding tooltip and help anchor pairs in a DHLP resource. You can load the help anchor with the DGGetDynamicHelpStrings function. After the help engine was launched with the help anchor parameter, DG frees the allocated memory by calling the BMKillPtr function. If contextHelpText is set to nullptr, DG tries to get the context help text from the GRC resource.

DGListViewUpdateData

If the image type of a list view item is DG_LVIT_OWNERDRAW and this item should be redrawn, DG sends a DG_MSG_UPDATE message to the dialog callback function. The message data parameter of this message is a pointer to a DGListViewUpdateData structure that contains all the necessary information to draw the image by the application.

    struct DGListViewUpdateData {
      void*  drawContext;
      short  listItem;
      short  width;
      short  height;
      short  left;
      short  top;
      short  filler;
    };

Members

drawContext
The drawing context of the image field. Cast this value to CGContextRef on Macintosh or HDC on Windows.
 
listItem
1-based index of the list view item
 
width
Width of the image field in pixels. On return: Width of the currently drawn image in pixels.
 
height
Height of the image field in pixels. On return: Height of the currently drawn image in pixels.
 
left
On return: Horizontal position of the currently drawn image in pixels, relative to the item rectangle.
 
top
On return: Vertical position of the currently drawn image in pixels, relative to the item rectangle.
 
filler
Reserved.

Remarks

The origin of the drawing context defined by the drawContext member is set to the upper left corner of the image rectangle of the list view item. Within this rectangle the application can draw the image anywhere and in arbitrary size.

The application should just draw the image. Graying and blending are handled by DG after the dialog callback function returns. From DG version 1.1.1, in order to blend the right area only, the application can set left and top and modify width and height members on return to refer the actual rectangle of the drawn image.

Requirements

Version: 2.2.0 or later
Header: DG.h

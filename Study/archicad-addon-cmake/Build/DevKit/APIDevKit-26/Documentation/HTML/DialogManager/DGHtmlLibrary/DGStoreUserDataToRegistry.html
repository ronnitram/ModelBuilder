DGStoreUserDataToRegistry

Strores the given userdata to the registry at Dialog Manager Custom Data key under the current Archicad key.

    void DGStoreUserDataToRegistry (
      const GS::UniString&  keyName,
      const void*           data,
      ULong                 size
    );

Parameters

keyName
The keyname of the registry (on Windows) or preferences (on Macintosh).
 
data
Userdata to be stored.
 
size
Output parameter. If data is set to nullptr, this parameter retrieves the size of the stored data.
 

Return Values

The size of the stored data is returned in the size parameter.

Remarks

Custom userdata in DG can be stored in system preferences (Macintosh) or in the system registry (Windows). This means that a given data can be stored and retrieved by the DGStoreUserDataToRegistry and DGRetrieveUserDataFromRegistry functions respectively.

Requirements

Version: DG 2.2.0 or later
Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

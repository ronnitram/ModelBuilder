DG_MSG_DRAGDROP

A drag and drop event occurs on a dialog or on a dialog item.

Message parameters

dialId
ID of the dialog.
 
item
1-based index of the dialog item.
 
msgData
Pointer to a DGDragDropMsgData structure cast to DGMessageData.

Return Values

The return value depends on the subMessage field of the DGDragDropMsgData.

submessage Value
DG_DRAG_STARTING 1, if the drag starting is accepted, 0 otherwise. When starting from an edit text control, 0 means accepted drag starting. If the message is not processed by the item is forwarded to its parent dialog.
DG_DRAG_START Not used.
DG_DRAG_END 1, if the message was processed by the application, 0 otherwise. If 0 is returned, DG calls the default drag and drop end-handler.
DG_DRAG_SETDELAYEDDATA Not used.
DG_DRAG_ENTER 1, if the message was processed by the application, 0 otherwise. At entering an edit control 0 means processed message (message not forwarded to parent dialog).
DG_DRAG_LEAVE Not used.
DG_DRAG_OVER 1, if the message was processed by the application, 0 otherwise. At entering an edit control 0 means processed message (message not forwarded to parent dialog).
DG_DRAG_DROP 1, if the message was processed by the application, 0 otherwise. At entering an edit control 0 means processed message (message not forwarded to parent dialog).

Remarks

The dialog callback function can receive this message only after it was enabled by DGEnableMessage. This message must be enabled for tab pages as well.

Requirements

Version: DG 2.0.0 or later
Header: DG.h

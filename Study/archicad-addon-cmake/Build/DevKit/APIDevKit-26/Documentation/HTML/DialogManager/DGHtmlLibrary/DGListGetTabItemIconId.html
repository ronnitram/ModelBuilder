DGListGetTabItemIconId

Retrieves the icon ID of a tabulator field of a list box item.

    short DGListGetTabItemIconId (
      short  dialId,
      short  item,
      short  listItem,
      short  tabIndex
    );

Parameters

dialId
ID of the dialog or tab page. DG_TOP_MODAL specifies the topmost modal dialog.
 
item
1-based index of the dialog item.
 
listItem
1-based index of the list item. DG_LIST_TOP specifies the first item of the list, DG_LIST_BOTTOM specifies the last item of the list.
 
tabIndex
1-based index of the tabulator field.

Return Values

The ID of the icon in the given tabulator field. DG_NO_ICON is returned if the field has no icon, if the icon was set by DGListSetTabItemIcon, or if the function parameters are incorrect.

Remarks

The content of list items can be divided into more columns. A DG listbox can be imagined as a two-dimensional list consisting of rows and columns. One column is called a "tabulator field", a "tabulator item" is the intersection of a tabulator field and a row. The names "tabulator field" and "tabulator item" are inherited from the previous versions of DG. In the earlier versions texts of columns could be separated within a list item by inserting tabulator ('\t') characters into its text. You can specify an icon and/or text to the tabulator item: see DGListSetTabItemIconId and DGListSetTabItemText for details.

A tabulator item can contain an icon and/or text also, but only one icon can appear in a tabulator item. In case of using an icon with text in a tabulator item, the item text can be set with the DGListSetTabItemText function. If a tabulator item contains an icon and text also, the icon will be positioned either at the beginning or at the end of the text according to the justification. (The DG_IS_LEFT justification is applied for the tabulator item containing an icon and text also, even if the DG_IS_CENTER justification was given.)

Call this routine only after you set the number of tabulator fields and their properties by DGListSetTabFieldCount and DGListSetTabFieldData.

To implement ontab check boxes an icon on the tabulator field can be set by DGListSetTabItemIconId. In this case the DG_MSG_CLICK messages should be handled in the dialog callback function. These special check boxes look much nicer than ordinary check box controls glued onto a tabulator field because they are transparent and sizeable. Possible icon IDs are:

Value Meaning
DG_LIST_CHECKEDICON Checked check box icon.
DG_LIST_GRAYCHECKICON Grayed check box icon.
DG_LIST_MIXEDCHECKICON Filled check box icon, used to indicate mixed or undefined selection.
DG_LIST_FRAMELESSCHECKEDICON Frameless check box icon.
DG_LIST_UNCHECKEDICON Unchecked check box icon.

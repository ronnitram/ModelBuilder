DGRegisterAdditionalHelpLocation

Registers an AddOn's own help engine location by it's MDID and path.

    bool DGRegisterAdditionalHelpLocation (
      Int32                 developerId,
      Int32                 localId,
      const GS::UniString*  localHelpPath,
      const GS::UniString*  localHelpFile,
      const GS::UniString*  helpWebAddress,
      const GS::UniString*  tmpPath,
      const GS::UniString*  helpParameters
    );

Parameters

developerId
Input parameter: The DeveloperID part of the AddOn's MDID.
 
localId
Input parameter: The LocalID part of the AddOn's MDID.
 
localHelpPath
Input parameter: Path to the help engine on local machine.
 
localHelpFile
Input parameter: Help engine file name (if nullptr assumed to be 'index.html').
 
helpWebAddress
Input parameter: URL of help web site (e.g http://help.graphisoft.com ) (only valid if localHelpPath is nullptr).
 
tmpPath
Input parameter: Path to temporary directory on local machine.
 
helpParameters
Input parameter: Additional parameters added to the end of the help URL.

Return Values

Value Meaning
true The specified helpPath was succesfully registered. There was a helpPath already registered with this MDID.
false The specified helpPath was succesfully registered. There wasn't any helpPath registered with this MDID.

If an AddOn has a userinterface, and it want's to provide context sensitive help information, it should register it's own help engine's path during initialisation. This is mandatory, because DG pass the help anchor information to the engine as a parameter, and the engine's responsibility to process the parameter and bring up the most context sensitive information. If there's no registered engine, DG wont't know where to pass the help anchor.

Respectively, you should unregister the helpPath during AddOn unload phase: DGUnregisterAdditionalHelpLocation.

Remarks

The help engine is assumed to be a HTML-based index.html root, and DG passes the help anchor parameter in the anchor variable to the engine.

Further information about the help anchor syntax rules and the GRC Specification of the Dialog Help format can be found here: GRC Specification of the Dialog Helps.

Requirements

Header: DG.h

Module: DGLib [MAC], DG.dll [WIN]

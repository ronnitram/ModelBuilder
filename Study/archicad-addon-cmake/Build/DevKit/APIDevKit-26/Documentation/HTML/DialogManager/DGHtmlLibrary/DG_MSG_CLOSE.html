DG_MSG_CLOSE

The dialog is about to be closed.

Message parameters

dialId
ID of the dialog.
 
item
Value Meaning
DG_ABORT The dialog has been aborted by DGAbortModalDialog.
DG_CLOSEBOX The modeless or palette dialog has been closed by DGModelessClose or by the user clicking on the close box.
Any positive number The application returns this value when handling the DG_MSG_CLICK or DG_MSG_NULL messages. In case of DG_MSG_CLICK messages this positive value is usually the 1-based index of the clicked push button.
 
msgData
Not used, always zero.

Return Values

Not used, should be set to zero.

Remarks

Before a dialog is in fact closed, DG always sends a DG_MSG_CLOSE message to the dialog callback function. The value of the item parameter refers to the method as the dialog has been closed. In case of modal dialogs the value of item will also be the return value of the DGModalDialog function.

When the dialog callback function receives the DG_MSG_CLOSE message its dialog items have not been destroyed yet. This means that they keep their values, states, etc. that the user set in the dialog. Use dialog item specific functions to retrieve these current settings. You can call DGModified to test whether a dialog item has been modified during the lifetime of the dialog or not.

If the application has allocated resources (images, memory blocks, etc.) to a dialog or to one or more of its items, it is advised to free them when handling the DG_MSG_CLOSE message.

Requirements

Version: DG 1.0.0 or later
Header: DG.h

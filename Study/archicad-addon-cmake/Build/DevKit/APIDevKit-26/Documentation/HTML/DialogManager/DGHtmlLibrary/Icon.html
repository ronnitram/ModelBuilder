Icon

Icon controls display masked images and they are generally used to label other controls in a dialog. Icon controls can also serve as background for other controls, i.e. it is possible to use one large explanatory icon for more focusable controls (for example edit controls) instead of using small separate icons.

By default an icon control is drawn without frame, however it can have three types of frames: static edge, client edge and modal frame.

Picture controls are very similar to icon controls, the only difference is that their images are not masked.

Item type

DG_ITM_ICONITEM

Frame types

Frame type Meaning
DG_FT_NONE No frame.
DG_FT_STATIC Thin three-dimensional border with a sunken edge.    
DG_FT_CLIENT Thick three-dimensional border with a sunken edge.    
DG_FT_MODAL Thick three-dimensional border with a raised edge.    

GRC Specification

    Icon    x  y  dx  dy  iconID  [frameType]

where x and y are the pixel coordinates of the upper left corner of the icon control, dx and dy are the width and height of the control in pixels. iconID defines the icon resource ID of the control. If the optional frameType flag is omitted, the icon is drawn without a frame.

iconID can be any positive resource ID or one of the following predefined constants:

Value Meaning Macintosh Windows
NoIcon No icon, empty space.
ErrorIcon Error icon.
InfoIcon Information icon.
WarningIcon Warning icon.

Possible values of frameType:

Value Meaning
Default The DG_FT_NONE frame type is set.
StaticEdge The DG_FT_STATIC frame type is set.
ClientEdge The DG_FT_CLIENT frame type is set.
ModalFrame The DG_FT_MODAL frame type is set.

Messages

Message Meaning
DG_MSG_CLICK The icon control is clicked. The message data parameter is always zero.
DG_MSG_DOUBLECLICK The icon control is double clicked. The message data parameter is always zero.

Remarks

If an icon control is used as explanatory background for other controls, it must precede them in the dialog item list. These controls can be fully or partly within the area of the icon control.

The icon of an icon control can be assigned either by the resource ID or by the handle of the icon. To set and retrieve the icon use DGSetItemImageId or DGSetItemImage and DGGetItemImageId or DGGetItemImage, respectively. Note that the icon resource ID can also be defined in the GRC script.

When an icon control is created or a new icon is assigned to it from program code, the control is automatically resized to the size of the icon. During this operation the center of the control is not moved and the possible frame is taken into account.

In order to spare with resource space, always use as few colors in the color table of the icon as possible. For example if your image contains 6 different colors only, do not use an icon with 256 colors, but use one with 16 colors instead.

DGInit

Initializes the DG dialog manager.

    bool DGInit (
      const DGInitData*  initData
    );

Parameters

initData
The DGInitData structure contains information about general DG settings and preferences.

Return Values

Value Meaning
true DG was successfully initialized.
false Some error occurred during initialization.

Remarks

This function must be called only once for every process, before any calls to DG functions. Before terminating the application, do not forget to call DGExit to free internally allocated system resources. If an application has some dynamically linking components (shared libraries on Macintosh, DLLs on Windows) it is forbidden to use DGInit or DGExit once again from the library. These components use the dialog manager initialized in the main program.

There is a version check in the DG dialog manager when calling the DGInit function. The relation between version numbers given in the DGInitData structure (external versions) and the ones in the DG dialog manager (internal versions) must be the following: mainVersion members must be equal, internal subVersion must be greater or equal than the external one, and if subVersions are equal the internal revision must be greater or equal than the external one. This procedure ensures that no older version of the DG dialog manager can be used.

Requirements

Version: DG 2.0.0 or later
Header: DG.h
Import Library: DG.lib [WIN]
Module: DGLib [MAC], DG.dll [WIN]

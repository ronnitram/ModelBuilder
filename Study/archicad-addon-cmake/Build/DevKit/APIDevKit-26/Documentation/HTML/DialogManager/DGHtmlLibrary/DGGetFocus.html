DGGetFocus

Retrieves the item index and dialog identifier of the item having the keyboard focus.

    void DGGetFocus (
      short   dialId,
      short*  focusDial,
      short*  focusItem
    );

Parameters

dialId
ID of the dialog. DG_TOP_MODAL specifies the topmost modal dialog. DG_ALL_DIALOGS also can be used here.
 
focusDial
Output parameter: Dialog or tab page identifier of the focused item. This parameter can be nullptr.
 
focusItem
Output parameter: 1-based index of the focused item. This parameter can be nullptr.

Remarks

If dialId is DG_ALL_DIALOGS then all the opened dialogs are searched for the focused item. Use this value to get the currently focused dialog and dialog item.

In contrast, call this function with a given dialog ID to test whether one of its items has the keyboard focus or not.

If the function parameters are incorrect, or the specified dialog does not have the keyboard focus or it is a tab page, zero returned for both focusDial and focusItem.

If either of the pointer parameters is nullptr, that value is not returned.

The keyboard focus and the focusable control categories in DG essentially have the same meanings as in the Macintosh operating system (i.e. the same type of controls are focusable in DG and on Macintosh). On Windows, however, more controls can have keyboard focus. Focusable controls of DG form a subset of focusable controls of Windows. The DGGetFocus and DGSetFocus functions work on focusable controls of DG only.

The following controls are focusable in DG: Date & Time Control, Edit Control, List Box, List View, Tree View.

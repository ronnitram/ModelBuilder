DG_MSG_RESIZE

The dialog is being resized.

Message parameters

dialId
ID of the dialog.
 
item
Not used, always zero.
 
msgData
Pointer to a DGResizeMsgData structure cast to DGMessageData.

Return Values

Not used, should be set to zero.

Remarks

DG_MSG_RESIZE messages are sent to the dialog callback function when a dialog or a tab page is resized. The major difference between the other messages and DG_MSG_RESIZE is that this message is sent not only by user action but also by program code. Moreover DG_MSG_RESIZE differs from the DG_MSG_GROW message in that it is sent to the tab pages also.

By every resizing action the dialog callback function receives three or more DG_MSG_RESIZE messages with the following submessages:

Value Meaning
DG_RSM_RESIZEENTER This message is sent before the dialog is resized.
DG_RSM_RESIZING This message is sent when the dialog is being resized.
DG_RSM_RESIZEEXIT This message is sent after the dialog is resized.

If a dialog contains one or more tab controls, tab pages receive the resize messages with submessages in the following order: First the DG_RSM_RESIZEENTER message is sent to the main dialog followed by a DG_RSM_RESIZING message. Responding to every DG_RSM_RESIZING message the application resizes its controls. When the tab control is resized, a DG_RSM_RESIZEENTER and a DG_RSM_RESIZING message is sent to every tab page of the tab control. If the resizing ends the tab pages receive DG_RSM_RESIZEEXIT first then at last the main dialog also.

 Windows only:

If the fulldrag system flag is on and the user begins to resize a dialog, the callback function permanently receives DG_MSG_RESIZE messages with DG_RSM_RESIZING submessage until the user finishes the resizing.

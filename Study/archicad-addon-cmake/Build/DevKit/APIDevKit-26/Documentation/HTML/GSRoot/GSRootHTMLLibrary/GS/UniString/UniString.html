Description

UniString is a Unicode string, that is sequence of Unicode characters (UniChars).
UniString has reference counted implementation, so strings can be passed and copied by value very fast because copies and their sources will share the same content until any of them are modified (this technique is called COW: Copy On Write). The reference counted implementation facilitates returning strings by value.
UniString stores Unicode characters in one continuous reallocating memory buffer. When a new character does not fit into the internal buffer a larger one is allocated and all characters are copied into the new buffer. Size of the new buffer will be 1.5 times larger than the old one. Reallocation may also happen when characters are deleted from the string to free the unused memory. This happens when less (or equal) than half of the buffer is used. In such cases size of the new buffer will be approximately as much large as is currently requeired.
Few memory buffers of recently destroyed smaller strings are cached and are reclaimed to new strings, so for example it is cheap to use (smaller) local strings inside methods and functions because their creation and destruction rarely involve memory heap operations. Note that empty strings do not use memory.
Strings, characters, substrings and C strings can be very effectively concatenated in one expression using the operator+ operation because the whole concatenation is performed in one step after all of the operands of the concatenation are gathered.
String has substring creating operations (eg. operator()). A substring refers to some part of the source string, so itself never allocates memory. Substrings can be effectively compared with various types (string, substring, C string etc) and can participate in various assigments and concatenations. These operations are always performed on the corresponding parts of the source strings.

ConstBidirectionalIterator Class

Iterator for bidirectional enumeration of sequence of constant items. 

template <class Type>
class ConstBidirectionalIterator : public ConstForwardIterator< Type >

Template parameters

Type
Type of items the iterator iterates on.

Members

Public Operators

operator++ Moves forward to the next item of the sequence.
operator-- Moves backward to the previous item of the sequence.

Description

A Contant Bidirectional Iterator is a constant iterator that can be both incremented and decremented. The requirement that a Constant Bidirectional Iterator can be decremented is the only thing that distinguishes Constant Bidirectional Iterators from Constant Forward Iterators. A constant bidirectional iterator returns a constant object when is dereferenced.

Iterators are created and returned by containers and than used to go through the elements of the container or they are passes as arguments to algorithm functions.

Most algorithms are expressed not in terms of a single iterator but in terms of a range of iterators the notation [first, last) refers to all of the iterators from first up to, but not including, last. Note that a range may be empty, i.e. first and last may be the same iterator. Note also that if there are n iterators in a range, then the notation [first, last) represents n+1 positions. This is crucial: algorithms that operate on n things frequently require n + 1 positions. Linear search, for example Find must be able to return some value to indicate that the search was unsuccessful.

Iterators are the mechanism that makes it possible to decouple algorithms from containers: algorithms are templates, and are parameterized by the type of iterator, so they are not restricted to a single type of container.

Example

GS::Array<GS::String> array;                      // array of strings
const GS::Array<GS::String>& constArray = array;  // constant array of strings

GS::Array<GS::String>::ConstIterator it = constArray.Enumerate ();    // obtaining a constant iterator (or constArray.Begin ())
while (it != NULL) {                    // until the end of sequence (constant array)
    GS::String s = *it;                 // dereferencing the constant iterator (returns constant reference)
    USize length = it->GetLength ();    // dereferencing the constant iterator (returns constant pointer)
    ++it;                               // advance to the next element
}


GS::Find (constArray.Begin (), constArray.End (), GS::String ("Test"));    // using constant iterators in algorithms

This example illustrates the use of constant iterators obtained from a constant container (constant array).

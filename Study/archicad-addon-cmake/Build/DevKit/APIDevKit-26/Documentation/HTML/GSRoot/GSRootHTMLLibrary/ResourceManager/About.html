About the Resource Manager

The Resource Manager provides basic resource management functionality.

The resources are identified by resource type and resource identifier. The resource type is a GSResType variable, the resource identifier must be a GSResID value.

 Windows only:
The resource type can be given by a character string, too.

A resource files can be opened by RSOpenResourceFile, which returns a resource module id (GSResModule), which can be used in the other resource manager functions to identify the resource module. The resource modules are put onto the resource module list. You can determine the active resource module by RSGetActiveModule, and modify it by RSSetActiveModule. The RSSearchResource function can be used to determine the module id of the resource module, which contains a resource given by its type and identifier. The resource data can be loaded by RSLoadResource into memory. When a resource file is not needed any more, it can be closed by RSCloseResourceFile.

Remarks:

The Resource Manager has only C++ interface, it can not be used from C code.

The types used in the Resource Manager are the following:

        typedef long            GSResModule;
        typedef unsigned long   GSResType;
        typedef long            GSResID;

        enum RSFlags {
                RSF_MissingResourceAlert        = 0x00000001
        };

        enum ResStringType {
                RST_Localised   = 0,
                RST_Fixed       = 1,
                RST_StatusBar   = 2
        };

        const GSResModule       AllResModule = -1;
        const GSResModule       AppResModule = -2;
        const GSResModule       SysResModule = -3;

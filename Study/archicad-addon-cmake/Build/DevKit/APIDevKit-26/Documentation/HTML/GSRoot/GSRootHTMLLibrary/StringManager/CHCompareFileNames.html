CHCompareFileNames

Compares two file name/paths using the specified character code.

    long CHCompareFileNames (
      const char*  p1CFileName,
      const char*  p2CFileName,
      GSCharCode   charCode = CC_System
    );

Parameters

p1CFileName
[in] The file name (C string) to be compared.
p2CFileName
[in] The file name (C string) to be compared with.
charCode=CC_System
[in] Specifies the character code used for the comparsion. See GSCharCode.

Return Values

Value Meaning
<0 The string pointed to by the p1CFileName parameter is less in lexical value than the string pointed to by the p2CFileName parameter.
0 The string pointed to by p1CFileName is equal in lexical value to the string pointed to by p1CFileName.
>0 The string pointed to by the p1CFileName parameter is less in lexical value than the string pointed to by the p2CFileName parameter.

Remarks

This function is a string comparsion, that is case sensitive on case sensitive file systems, and non case insensitive, on case insensitive file systems, such as Windows and MacOS.

Notice that if the return value is 0, the two strings represents the same file, though not necessarily identical.

Note that if the two string represents paths, the function might return non 0 value, though they represent the same file/directory. Eg. Comparing CHCompareFileNames("etc/../..","..") != 0, while CHCompareFileNames("test.txt","Test.txt")==0 if the file system is case insensitive.

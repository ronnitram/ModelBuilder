CHANSI2Unicode

Converts an ANSI string to unicode string.

    long CHANSI2Unicode (
      const char*  pChars,                    // string to convert
      long         nChars,                    // number of bytes in string
      wchar_t*     pWChars,                   // wide-char buffer
      long         maxWChars,                 // wide char buffer length
      GSCharCode   charCode = CC_Default        // Character codepage of input
      CHANIS2UnicodeCacheRef*  cache = NULL           // can be used to cache conversion result, if called twice
    );

Parameters

pChars
[in] Points to the charater string to be converted.
nChars
[in] Specifies the size in bytes of the string pointed to by pChars parameter, or it can be CStringLen if the string is null terminated.

If this string is null terminated, the resulting wide character string has a null terminator, and the returned length includes the null terminator.

If this parameter is a positive integer, the function processes exactly the specified number of bytes. If the given length does not include a null terminator then the resulting wide character string will not be null terminated, and the returned length does not include a null terminator.

pWChars
[out] Points to a buffer that receives the translated string.

Can be NULL only if maxWChars is a null pointer.

maxWChars
[in] Specifies the size, in wide characters, of the buffer pointed to by the pWChars parameter. If this value is zero, the function returns the required buffer size, in wide characters, and makes no use of the pWChars buffer.

See Remarks for implementations limitation regarding this parameter in Archicad 7.0 R1/V1

charCode=CC_Default
[in] Specifies the code page to be used to perform the conversion. See GSCharCode for more information. CC_UTF8 is valid for this parameter.
cache
[in/out] If not null caches the conversion result for two-pase calls. Typically you call the conversion twice with the same ansi string. First to calculate the lenght of the result, so you can allocate a buffer, then to store a result in the buffer. You can use this parameter to cache the converted string used for lenght calculation so it can be reused at the conversion. If maxChars is zero and cache is not NULL, it should point to a NULL ANIS2UnicodeCacheRef. If there's any reusable infomration it will be stored in this reference. The reference is freed by either the next call to CHANSI2Unicode, or CHFreeANSI2UnicodeCache. If maxChars is non zero and cache is not NULL, and points to a valid cache, the conversion uses this cache to help the conversion. In case the cache is non-zero the function might use the cache data instead of pWChars and nWChars. The cache will be released, and zeroed after it's use. 
GS::ANIS2UnicodeCacheRef cache = 0;

len = CHANSI2Unicode (ansiStr, GS::CStringLen, 0, 0, CC_Default, true, &cache);

if (len > 0) {

  try {

    p = new GS::uchar_t [len];

    res = CHANSI2Unicode (ansiStr, GS::CStringLen, p, len, CC_Default, true, &cache);

  } catch (std::bad_alloc& ) {

    if (cache)

      CHFreeANSI2UnicodeCache (cache);

  }

}

Return Values

If the function succeeds, and maxWChars is nonzero, the return value is the number of wide characters written to the buffer pointed to by pWChars.

If the function succeeds, and maxWChars is zero, the return value is the required size, in wide characters, for a buffer that can receive the translated string.

If the function fails, the return value is zero. This function can fail if:

-Not enough memory.

-Invalid falgs/parameters.

-No unicode translation is present.

-The provided buffer is too small.

Remarks

The pChars and pWChars pointers must not be the same, or overlap. If they do the result of this function is undefined!

The unrepresentable characters are translated to system dependent default replacement character on Windows and to '?' on MacOS.

This function is implemented using CoreFoundation String services on MacOS, and MultiByteToWideChar on Windows.

RSGetIndString

Copies a string from a string list resource to the memory.

    bool RSGetIndString (
      char*          pText,
      GSResID        resID,
      long           index,
      GSResModule    resModule = AllResModule,
      ResStringType  rsType = RST_Localised
    );

Parameters

pText

[out] A pointer to a character buffer for holding the string value.

The size of the buffer should be large enough to the string value.The maximum size of a such string is 256 on Macintosh, there's no such maximum value in Windows, therefore you should provide a large enough buffer..

resID
[in] A numeric resource identifier.
index
[in] The index of the string in the list. This is a value indexed from 0.
resModule=AllResModule
[in] The resource module to look for the string list in. A resource module to get the resource from. If this value is AllResModule then all resource modules opened with RSOpenResourceFile will be searched.
rsType=RST_Localised
[in] The type of the resource string. It can be: RST_Localised,RST_Fixed and RST_StatusBar

Return Values

True if the load succeeded.

Remarks

You can use the GRC resource preprocessor to generate cross platform string lists.

If the requested string list resource is not found, the debug version of the library will break into the debugger. Use RSMissingResourceAlert to change this behaviour.

Requirements

Version: 1.3 or later
Header: RS.hpp
Import Library: Link with GSRootImp.lib
Module: GSRootLib [MAC], GSRoot.dll [WIN]

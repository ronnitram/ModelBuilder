Description

IChannel is an abstract base class representing sequential data source (input). It can be thought of as possibly infinite sequence of characters (character stream). After a character is consumed, the next one becomes available for the next read (access) operation.
IChannel can be used on two different levels.

The lower level is the binary (or character) level, where there is no any conversion on the input, it is passed to the user as it was read from the data source behind the input channel.

The higher level is the typed level, where simple C++ types (and objects) can be read from the input channel. Methods belonging to this level perform the read operation through the IProtocol (input protocol) object which is currently selected into the input channel. When the input channel is invoked to read a particular type, it passes that request to its current IProtocol object, which uses the binary level of the channel to obtain new data, from which it composes the requested type and returns it to the channel which will return it to the caller.
This indirection allows user to use the same code over the input channel without regard to the real representation of simple C++ types on the data source, that is, user should always read shorts or doubles from the input channel in the same way after the appropriate input protocol has been set. Request to read an object is also passed through the input protocol.
Input protocols may change the byte order or may interpret and convert text representation of the data etc. They know how the particular type should be composed from its real representation. Every input channel should have a default IProtocol object.

To make the error handling easier, input channel stores the input status. This status is NoError until an error occures. After that it stores the error code (e.g. EndOfInput or ReadError) until the input status is reseted by the user. After an error has been occurred, methods performing read-like operations do nothing and immediately return with the input status. By reseting the input status, these methods will behave normally and will try the reading again.

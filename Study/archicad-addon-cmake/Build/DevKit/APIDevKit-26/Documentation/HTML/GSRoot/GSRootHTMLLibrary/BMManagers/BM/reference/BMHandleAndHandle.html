BMHandleAndHandle

The BMHandleAndHandle function concatenates two relocatable memory blocks.

GSErr BMHandleAndHandle(
  GSHandle        inpH,  // input handle
  GSHandle        outH   // output handle
);

 

Parameters

inpH
Specifies the handle of the first relocatable memory block, of which contents do not change but concatenated to the end of the second relocatable block. It can not be NULL and it must be a nonempty handle.
outH
Specifies the handle of the second relocatable block, of which size the BM Manager expands in order toconcatenate the data from inpH to the end of the contents of this block. It can not be NULL.

Return Values

If the function succeeds, the return value is 0 (NoError).

If the function fails, the return value is a Graphisoft error code. To get extended error information, call BMError.

If the inpH or outH parameters is NULL, the return value is ErrNilHandle.

If the inpH parameter specifies an empty handle, the return value is ErrEmptyHandle.

Requirements

    Header: Declared in BM.hpp
    Import Library: Link with GSRootImp.lib

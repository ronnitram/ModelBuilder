OChannel::Flush

Flushes internal buffers to the data destination. 

virtual GSErrCode Flush (void);

Return Value

Value Meaning
NoError* OK
OutputIsFull* output device is full
WriteError* write error on the output device
Error* unspecified error

* declared in OChannel class scope
* declared in GS namespace, but accessible in the global scope

Description

Output channel implementations may buffer the data written by the user before it is really sent to the data destination (e.g. for efficiency reasons). If that is the case then this method instructs the output channel to flush internal buffers to the data destination. However flushing usually can not be performed when marks are put (with the PushMark method) or when the write pointer is returned with the PopMark method and is not yet reseted with the ResetOffset method because in such cases data must usually reside in memory.
Output status will record and hold the first error occurred on the output channel. After an error this method does nothing and immediately returns with the stored error code (until the output status is reseted by the user).

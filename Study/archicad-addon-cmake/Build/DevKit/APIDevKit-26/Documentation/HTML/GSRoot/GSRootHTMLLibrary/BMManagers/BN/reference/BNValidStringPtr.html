BNValidStringPtr

The BNValidStringPtr function verifies that the calling process has read access to a range of memory pointed to by a string pointer.

bool BNValidStringPtr(
  const char      *strPtr,               // address of string
  GSSize          maxSize                // maximum size of string
);

 

Parameters

strPtr
Pointer to the a null-terminated string.
maxSize
Specifies in characters the maximum size of the string. The function checks for read access for all bytes up to the string's terminating null character or up to the number of bytes specified by this maxSize parameter, whichever is smaller. If this parameter is zero, the return value is true.

Return Values

Macintosh:
The return value is always true.

Windows:
If the calling process has read access to all bytes up to the string's terminating null character or up to the number of bytes specified by maxSize, the return value is true.

If the calling process does not have read access to all bytes up to the string's terminating null character or up to the number of bytes specified by maxSize, the return value is false.

Remarks

Windows:
If the application is compiled as a debugging version, and the process does not have read access to all bytes in the specified memory range, the function causes an assertion fault and breaks into the debugger. Leaving the debugger, the function continues as usual, and returns true. This is a designed behavior, its main purpose is to provide a debugging aid.

If the calling process has read access to some, but not all of the bytes in the specified memory range, the return value is false.

The BNValidStringPtr function wraps the IsBadStringPtr Win32 function.

Requirements

    Import Library: Link with GSRootImp.lib

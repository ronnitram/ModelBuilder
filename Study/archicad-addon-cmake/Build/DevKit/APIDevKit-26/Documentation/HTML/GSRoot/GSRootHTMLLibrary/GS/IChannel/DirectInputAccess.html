IChannel::DirectInputAccess

Makes content of the input channel directly accessible. 

virtual GSErrCode DirectInputAccess (
    USize               nOfCharToAccess,
    char **             source,
    USize *             accessible = NULL
) = 0;

Parameters

nOfCharToAccess
Number of characters to be accessed. If this value is 0 (desirable) then the input channel will calculate and offer an optimal size in the accessible parameter.
source
On return: pointer to the accessible data on the input channel. This pointer will remain valid until the next operation on the input channel is invoked.
accessible
On return: if not NULL then number of accessible characters.

Return Value

Value Meaning
NoError* OK
EndOfInput* end of input has been reached, operation is not completed
ReadError* read error on the input device, operation is not completed
ErrMemoryFull* out of memory
Error* unspecified error

* declared in IChannel class scope
* declared in GS namespace, but accessible in the global scope

Description

This method provides fast direct access to content of the input channel. User can specify number of characters to be directly accessed or can let the input channel determine an optimal size (desirable). The resulting pointer can be used to directly access (or even to manipulate) subsequent part of the input channel's content. This pointer will reamin valid until the next operation on the input channel is invoked. When the processing is completed data can be skipped with the Skip method.
If there is no enough data left on the input channel then EndOfInput is returned. In such cases the output accessible parameter contains the number of accessible characters.
Input status will record and hold the first error occurred on the input channel. After an error this method does nothing and immediately returns with the stored error code (until the input status is reseted by the user).

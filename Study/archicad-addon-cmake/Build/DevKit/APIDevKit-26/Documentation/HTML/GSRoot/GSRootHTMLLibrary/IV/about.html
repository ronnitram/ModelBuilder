About IV

The IV provides functions to swap between the endiannes of primitive types like int, and doubles.

The PPC MacOS platform is big-endian, while the X86 Win32 and the Intel Macintosh platforms are little-endian. When reading binary files from a different platform you have to call the IV functions.

Example:

When you write a short 255 (=0x00FF) on PPC Macintosh, and read it back on Windows or on a MacTel machine without swapping the result will be 65280 (=0xFF00). So when you're reading a file that contains short-s coming from a different platform you should call IVShort -to swap the bytes.

Description

Random containers are containers where an unique integer index (position) is associated with every item storage cell in the container beginning from 0 and continuously increasing by 1. Storage cells are created and used one by one from the beginning when items are put into the random container. Consequently the last index is the size of the random container minus 1 (if it is not empty). Items are directly accessible (referrable) with these indices. Due to these integer indices random containers have ordering (they have unambigous first, next to, previous to and last items).
Note that indices are associated with storage cells and not with the items themselves. So when an item is inserted into (or deleted from) the middle of the random container, indices of items above it will increase (decrease) by one.

GSPref::SetLongValue, GSPref::SetBoolValue, GSPref::SetStringValue, GSPref::SetANSIStringValue

Writes a typed data to the registry on Windows, preference file on MacOS .

    bool SetLongValue (
      RootKeyRef  root,
      const char*   path,
      const char*    valueName,
      const long    valueData
    );

    bool SetBoolValue (
      RootKeyRef  root,
      const char*   path,
      const char*    valueName,
      const long    valueData
    );

    bool SetStringValue (
      RootKeyRef  root,
      const char*   path,
      const char*    valueName,
      const wchar_t*    valueData
    );

    bool SetANSIStringValue (
      RootKeyRef  root,
      const char*   path,
      const char*    valueName,
      const GSCharCode    valueCharCode,
      const char*        valueData
    );

Parameters

root
[in]Pointer to the root key, created with CreateRootKey. See RootKeyRef.
path
[in] Pointer to a string containing a '/' separated path. The value is written to the sub-key specified by this path relative to the root. If the sub-key defined by this path is not already present it is created. Use "" if you want to write direcly under the root key.

The path name cannot begin or end with a '/'. The size of the path cannot exceed 1024 characters, and cannot be NULL. It can contain only ASCII characters.

valueName
[in] Pointer to a string containing the name of the value to set. If a value with this name is not already present, the function creates it.

The valueName cannot contain '/' characters, and it should contain only ASCII characters. The size of the name cannot exceed 255 characters.

valueCharCode
SetANSIStringValue [in] The GSCharCode of the parameter valueData
valueData
[in] The data to be stored with the specific value name.

SetLongValue A 32-bit signed integer to be stored.

SetBoolValue A bool value.

SetStringValue An unicode encoded, null terminated string value. The maximum length of the string with null terminator is 2048 bytes. If the length is more than 2048 bytes the result is undefined.

SetANSIStringValue A char-code encoded null terminated single or multibyte string. The string is converted to unicode based on the valueCharCode param, and the unicode value is stored. The maximum length of the string is 2048 bytes. If the length is more than 2048 bytes the result is undefined.

Return Values

True if the function succeeds false otherwise. If the function fail the registry/preference data remain unchanged.

This function can fail in the following cases:
  • Not enough resources. (Memory allocation error, etc.)
  • Not enough rights to write the registry.
  • In case of SetANSIStringValue the ANSIToUnicode conversion failed.

Remarks

On MacOS the value names is in the same namespace as the directory names. On Windows the value and directory namespace are different. Therefore you shouldn't write to a value/direcory with the same name as an existing directory/value, because on MacOS the existing directory/value will be replaced by the new value/directory.

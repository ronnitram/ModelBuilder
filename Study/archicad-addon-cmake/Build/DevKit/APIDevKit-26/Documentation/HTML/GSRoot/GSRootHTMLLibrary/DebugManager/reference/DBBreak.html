DBBreak

The DBBreak function performs a debug break: throws a debug string (containing the name of the source file and the line number and an optional message string) and breaks into the debugger. 

bool DBBreak(
  const char    *fileName, // name of the source file
  unsigned long *lineCount,        // current line number
  const char    *pMessage     // optional message
);

 

Parameters

fileName
Specifies the name of the source file where the break occurs.
lineCount
Specifies the line number exactly where the break occurs.
pMessage
Specifies the message. This parameter is optional, i.e. NULL is allowed.

Return Value

The return value is always true.

Remarks: 

Behavior of the BDBreak function is platform-dependent. 

Windows:
DBBreak displays an assertion message box where you can choose to break into the debugger or not.

DBBreak uses the DebugBreak Win32 function to break into the debugger. DebugBreak raises a breakpoint exception in the current process that is, the calling thread can awake the debugger and force it to take some action. If the process is not under debug, the search logic of a standard exception handler is used. In most cases, this causes the calling process to terminate, throwing an unhandled breakpoint exception.

Macintosh:
DBBreak does not break into the debugger only sends a debug string.

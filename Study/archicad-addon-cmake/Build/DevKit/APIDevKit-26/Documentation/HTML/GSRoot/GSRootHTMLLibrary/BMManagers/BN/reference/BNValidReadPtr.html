BNValidReadPtr

The BNValidReadPtr function verifies that the calling process has read access to the specified range of memory.

bool BNValidReadPtr(
  const void      *memPtr,               // pointer to memory range
  GSSize          len    // length of memory range
);

 

Parameters

memPtr
Pointer to the memory range to verify.
len
Specifies the length of the memory range to verify.

Return Values

Macintosh:
The return value is always true.

Windows:
If the calling process has read access to all bytes in the specified memory range, the return value is true.

If the calling process doesn't have read access to all bytes in the specified memory range, the return value is false.

Remarks

Windows:
If the application is compiled as a debugging version, and the process does not have read access to all bytes in the specified memory range, the function causes an assertion fault and breaks into the debugger. Leaving the debugger, the function continues as usual, and returns true. This is a designed behavior, its main purpose is to provide a debugging aid.

If the calling process has read access to some, but not all of the bytes in the specified memory range, the return value is false as well.

The BNValidReadPtr function wraps the IsBadReadPtr Win32 function.

Requirements

    Import Library: Link with GSRootImp.lib

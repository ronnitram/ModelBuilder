Description

OChannel is an abstract base class representing sequential data destination (output). It can be thought of as possibly infinite sequence of characters (character stream). After a character is written, it becomes the last character of the sequence.
OChannel can be used on two different levels.

The lower level is the binary (or character) level, where there is no any conversion on the output, it is passed to the data destination behind the output channel as it was written by the user.

The higher level is the typed level, where simple C++ types (and objects) can be written to the output channel. Methods belonging to this level perform the write operation through the OProtocol (output protocol) object which is currently selected into the output channel. When the output channel is invoked to write a particular type, it passes that request to its current OProtocol object, which decomposes the type to its real representation and uses the binary level of the channel to write it to the data destination.
This indirection allows user to use the same code over the output channel without regard to the real representation of simple C++ types on the data destination, that is, user should always write shorts or doubles to the output channel in the same way after the appropriate output protocol has been set. Request to write an object is also passed through the output protocol.
Output protocols may change the byte order of the data or may convert it to some text representation etc. They know how the particular type should be decomposed to its real representation. Every output channel should have a default OProtocol object.

To alleviate the fully sequential nature of the output channel and to allow rewriting of the previously issued data, so-cold marking is introduced. User can mark the current position on the output channel, continue with writing and return to this position later to rewrite the old data. When the write pointer is returned, offset from the end of the output stream can be queried, and finally reseted to jump back to the end of the output stream. Marking can be nested.

To make the error handling easier, output channel stores the output status. This status is NoError until an error occures. After that it stores the error code (e.g. OutputIsFull or WriteError) until the output status is reseted by the user. After an error has been occurred, methods performing write-like operations do nothing and immediately return with the output status. By reseting the output status, these methods will behave normally and will try the writing again.

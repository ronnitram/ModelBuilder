API Style GuideVersion: 1.0

Quality

Always handle errors and warnings coming from the API (it might be a bug in the server application :))!

Test for memory leaks (e.g. by using BoundsChecker in Visual C++, or SpotLight on the Macintosh)!

Test on every possible operating system you can get hold of (Windows 2000 or XP; MacOS X 10.2.3-), also be aware of two-byte systems!

Be aware of teamwork mode! Not all the functionality works the same way. For example, elements you would like to work on can be in someone else's workspace; or a layer you would like to place an element to does not belong to the current user.

Report the bugs you found in the API immediately to through the developer support database!

Platform independence

DO use the functionality provided by the API and the modules in it: the Dialog Manager and GSRoot (string, memory management etc.). They are robust and well tested on both platforms.

  • DG is a platform-independent dialog manager, providing many different dialog and dialog item types. It also includes the platform-independent resource format called GRC. All of Archicad's dialogs (except for file and print dialogs) were created with it.
  • GSRoot provides string, memory, date and time, and timer management, generic container classes, and includes general debugging services on both platforms. You can also use the byte swapping functions to convert data between Mac and Windows.

DO use standard C and C++ calls. You can also use STL if you want, though some standard containers are now supplied by the GSRoot module.

DO use the Location class to access/store file/folder information. It can be mapped to platform-specific structures when needed.

Leave place for byte swapping (Mac and Win machines store the basic types in different byte orders). If you plan to do cross-platform reading/writing then you have to write the swapping routines yourself.

Take into account the internal alignment of the structures. Different compilers may align the internals of structures differently. Always align those which may go into a file/preferences to 8 bytes internally; see the structures in the API for examples.

Use application frameworks with care; they haven't been tested with the API.
